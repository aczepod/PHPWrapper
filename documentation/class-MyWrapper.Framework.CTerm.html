Class CTerm

Term object ancestor

A term can be compared to a word of a vocabulary, it is an elementary component that is an abstract concept if considered by itself, but takes a precise meaning if related to other terms, as a sequence of words that constitute a statement.

Terms feature a namespace, kOFFSET_NAMESPACE, which contains the native unique identifier, kOFFSET_NID, of the object that represents the namespace of the current term. This namespace may be provided as a kOFFSET_NID or as a full object in case it should be committed.

The global unique identifier, kOFFSET_GID, is the code by which the term is known outside of this system, it is constituted by the concatenation of the kOFFSET_GID of the namespace referred to in the kOFFSET_NAMESPACE offset, and the local unique identifier, kOFFSET_LID, of the current term, both separated by the kTOKEN_NAMESPACE_SEPARATOR token.

If the term has no namespace, its kOFFSET_GID will be its kOFFSET_LID.

The class features an attribute, kOFFSET_REFS_NAMESPACE, which keeps track of the number of times that the term has been referenced as a namespace, this offset is automatically managed bi the class.

When MyWrapper\Framework\CStatusDocument::_IsCommitted(), besides the the identifier offsets, also the kOFFSET_NAMESPACE offset will be locked, since it is used to generate the kOFFSET_GID.

The native unique identifier, kOFFSET_NID, is the binary md5 hash of the global unique identifier, kOFFSET_GID, this to allow long codes.

The object will have its MyWrapper\Framework\CStatusDocument::_IsInited() status set if the local unique identifier, kOFFSET_LID, is set, derived classes may add other required attributes.

Finally, the class features member accessor methods for the default offsets:

  • MyWrapper\Framework\CTerm::NS(): This method manages the term's namespace, kOFFSET_NAMESPACE.
  • MyWrapper\Framework\CTerm::LID(): This method manages the local unique identifier, kOFFSET_LID.
  • MyWrapper\Framework\CTerm::NamespaceRefs(): This method returns the term's namespace references, kOFFSET_REFS_NAMESPACE.

Manage namespace native identifier

Manage namespace native identifier

The namespace native identifier, kOFFSET_NAMESPACE, holds the native identifier of the object that represents the term's namespace.

The method accepts a parameter which represents either the namespace or the requested operation, depending on its value:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • other: Set the value with the provided parameter.

The second parameter is a boolean which if TRUE will return the old value when replacing containers; if FALSE, it will return the currently set value.

Note that when the object has the MyWrapper\Framework\CStatusDocument::_IsCommitted() status this offset will be locked and an exception will be raised.

# _index( MyWrapper\Framework\CContainer $theContainer, bitfield $theModifiers )

Return the object's global unique identifier

Return the object's global unique identifier

Term identifiers are constituted by the concatenation of the global unique identifier of the namespace term and the local unique identifier of the current term separated by a kTOKEN_NAMESPACE_SEPARATOR token.

If the term lacks a namespace, its global identifier will be its local identifier.

This method will take care of getting the global identifier from the eventual namespace term and generate the current term global identifier.

If the namespace was set in the form of an object, it will be converted back to its native identifier.

This method does not assume the type of object that it may find in the namespace, it will first check if derives from MyWrapper\Persistence\CPersistentObject, in that case it will use its global identifier; any other type will be assumed to be the native identifier of the namespace. In the latter case the method will instantiate the object and use its global identifier.
