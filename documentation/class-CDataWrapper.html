Class CDataWrapper

Data wrapper

This class overloads its ancestor to implement the framework for providing services that store and request data. The class does not feature a concrete data store engine, so it only concentrates in providing the framework for derived classes that implement concrete data store instances, for this reason this class is abstract.

These new functionalities require an additional set of parameters:

  • Request parameters: These parameters refer to the parameters that the service expects:
    • kAPI_SERVER: Server, this offset tag will be used to locate the service CServer object, concrete derived classes should instantiate this object in their CWrapper::_InitResources() method and store it in the $_REQUEST global array.
    • kAPI_DATABASE: Database, this string parameter represents the name of the database which is a concrete instance of CDatabase.
    • kAPI_CONTAINER: Container, this string parameter represents the name of the container within the provided database,it will be a concrete instance of CContainer.
    • kAPI_PAGE_START: Page start, this integer parameter represents the start offset for paging, it is a zero-based index.
    • kAPI_PAGE_LIMIT: Page limit, this integer parameter represents the maximum number of records to be returned.
    • kAPI_QUERY: Query, this array parameter represents the data query, it should follow the rules of the CQuery class family.
    • kAPI_SELECT: Select, this array parameter represents the query selection or fields list, it is constituted by an array of strings that indicate which fields should be returned by the query.
    • kAPI_SORT: Sort, this array parameter represents the list of fields upon which to sort the results, the array element indexes represent the field references, the values are provided as numbers where negative values represent a descending sense and zero or positive values ascending.
    • kAPI_DISTINCT: Distinct, this string represents the property name of an object: if provided, all queries will no longer return the list of matching elements, but the list of distinct values of that property of the query selection.
    • kAPI_OBJECT: Object, this array parameter represents the object that is to be inserted or updated in a container.
    • kAPI_CLASS: Class, this string parameter represents the class of the kAPI_OBJECT object parameter, if provided, it is assumed that the object provided in the kAPI_OBJECT parameter is an instance of that class.
  • Response parameters: These parameters refer to the parameters that the service is expected to return:
    • kAPI_PAGING: Paging, this offset is constituted by the list of parameters relating to paging:
      • kAPI_PAGE_START: Page start (input).
      • kAPI_PAGE_LIMIT: Page limit (input).
      • kAPI_PAGE_COUNT: Page count (output), the actual number of elements returned, which will be less or equal to the kAPI_PAGE_LIMIT parameter.
    • kAPI_STATUS: Status, this block features an additional element that is used by query requests:
      • kAPI_AFFECTED_COUNT: Affected count (output): the total number of elements affected by the operation.
      • kTERM_STATUS_IDENTIFIER: Object identifier (output): this parameter will contain the native identifier, kTAG_NID, of the newly created object when calling a kAPI_OP_INSERT operation. This is especially useful when that attribute is computed, rather than explicitly set beforehand.
  • Operations: This class adds the following operations:
    • kAPI_OP_COUNT: Count, this operation will return the total number of elements that satisfy the provided kAPI_QUERY; if the latter is omitted, it is assumed that we want the total number of elements in the kAPI_CONTAINER. The operation expects the following parameters:
      • kAPI_FORMAT: Format, this parameter is required, since other non scalar parameters must be encoded.
      • kAPI_DATABASE: Database, database to which the container belongs.
      • kAPI_CONTAINER: Container, container in which to perform the query.
      • kAPI_QUERY: Query (optional), the selection criteria.
    • kAPI_OP_GET: Query, this operation will return all records that satisfy the provided query. The operation expects the following parameters:
      • kAPI_FORMAT: Format, this parameter is required, since other non scalar parameters must be encoded.
      • kAPI_DATABASE: Database, database to which the container belongs.
      • kAPI_CONTAINER: Container, container in which to perform the query.
      • kAPI_QUERY: Query (optional), the selection criteria.
      • kAPI_SELECT: Select (optional), the list of fields to be returned.
      • kAPI_SORT: Sort (optional), the list of sort elements and their sense.
      • kAPI_PAGE_LIMIT:This parameter is required or enforced, it represents the maximum number of elements that the query should return, the default value is kDEFAULT_LIMIT.
      • kAPI_PAGE_LIMIT:This parameter is required or enforced, it
    • kAPI_OP_RESOLVE: Resolve, this operation will return the object that matches the provided value. The operation expects the following parameters:
      • kAPI_FORMAT: This parameter is required to indicate how to encode the response.
      • kAPI_DATABASE: This parameter is required to indicate the working database.
      • kAPI_CONTAINER: This parameter is required to indicate the working container. Note that in some cases this parameter is not required, in particular, when providing the kAPI_CLASS parameter, the container might be implicit.
      • kAPI_OBJECT: This parameter is required and contains the value to be matched.
      • kAPI_CLASS: This parameter is required, it represents the class from which the Resolve() static method will be used.
    • kAPI_OP_INSERT: Insert, this operation will insert the provided object into the database or container. The operation expects the following parameters:
      • kAPI_FORMAT: This parameter is required to indicate how to encode the response.
      • kAPI_DATABASE: This parameter is required to indicate the working database.
      • kAPI_CONTAINER: This parameter is required to indicate the working container. Note that in some cases this parameter is not required, in particular, when providing the kAPI_CLASS parameter, the container might be implicit.
      • kAPI_OBJECT: This parameter is required and contains an array or corresponding to the new record.
      • kAPI_CLASS: If provided, this parameter indicates which instance the object should be; if not provided, the kAPI_CONTAINER parameter is required and the object will simply be added to the container.
    • kAPI_OP_DELETE: Delete, this operation will delete either a selection of objects matching a query or one object matching a value. The operation expects the following parameters:
      • kAPI_FORMAT: This parameter is required to indicate how to encode the response.
      • kAPI_DATABASE: This parameter is required to indicate the working database.
      • kAPI_CONTAINER: This parameter is required to indicate the working container. Note that in some cases this parameter is not required, in particular, when providing the kAPI_CLASS parameter, the container might be implicit.
      • kAPI_QUERY: This parameter contains the query that selects the items to be deleted.
      • kAPI_OBJECT: This parameter is required and contains the value by which the object will be identified, some classes feature a Resolve() method, in that case this value will be handled by that method, if not, the value is assumed to be the object0s kTAG_NID. If this parameter is provided, the kAPI_QUERY parameter will be ignored.
      • kAPI_CLASS: If provided, this parameter indicates to which class the object to be deleted belongs to; if not provided, the kAPI_CONTAINER parameter is required and the object is assumed to be the object's kTAG_NID. This parameter is ignored if the kAPI_OBJECT parameter is missing.

Format query.

Format query.

The main duty of this method is to format the provided query. This parameter will be provided as an array in which the first level element indexes can take the following values:

# _HandleQuery( mixed $theResult = NULL, boolean $isCount = FALSE, boolean $doCount = TRUE )

Perform the current query.

Perform the current query.

This method will execute the query provided in kAPI_QUERY, selecting the fields provided in kAPI_SELECT, sorted by the fields provided in kAPI_SORT, starting at the page provided in kAPI_PAGE_START, counting the number of records provided in kAPI_PAGE_LIMIT.

The method expects two parameters:

  • $theResult: This parameter indicates the kind of query, it can take the following values:
    • NULL: Return the selection cursor (default).
    • TRUE: Return first matched record or NULL if there were no matches.
    • other: Any other value will be interpreted as a field identifier: in this case the method will return an array containing the distinct values of the provided field identifier.
  • $isCount: This boolean parameter indicates whether or not the query is intended for a count operation: TRUE means that the method will not return any result, except for the count information in the response parameters.
  • $doCount: This boolean parameter indicates whether or not the query should update the count and paging values of the request, if the parameter is TRUE, the default value, the query is considered as the final operation and all count and paging parameters will be updated accordingly; if the parameter is FALSE, the method will only perform the query and return the result.

The method will traverse query lists until a match is found and it expects the kAPI_DATABASE parameter to be set.

# _ValidateQueries( reference & $theQuery )

Validate query parameters.

Validate query parameters.

The duty of this method is to verify the provided query and to convert it into a native query object. The provided query may be of three kinds:

  • Scalar query: A single query.
  • Query list: A list of queries.
  • Container query list: A list of queries each using a different container.

The method will traverse the provided structure until it finds a single query with which it will perform the following steps:

  • Instantiate query: The query will be instantiated as a native CQuery derived instance.
  • Validate query: The structural integrity of the query will be validated (CQuery::Validate()).
  • Unserialise query: All data elements in the query which need to be converted to the native container format will be unserialised.

Depending on the query kind, the method expects the following service parameters to be provided:

  • Scalar query: The {@link kAPI_CONTAINER must be set.
  • Query list: The kAPI_CONTAINER must be set.
  • Container query list: The kAPI_DATABASE must be set.
