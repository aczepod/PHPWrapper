Class CDataWrapper

Data wrapper.

This class overloads its ancestor to implement a web service that wraps a data store, it represents a framework for building concrete data store web-service wrappers.

The class introduces a series of new operations and filter options that must be implemented in derived classes which implement a specific data store.

These new functionalities require a new set of parameters:

  • Data store parameters: In order to refer to a specific data store we need two parameters:
    • kAPI_DATABASE: Database, this parameter should indicate the database or equivalent concept where the data should be stored or retrieved. This parameter can be compared to the database part of an SQL table reference (DATABASE.TABLE).
    • kAPI_CONTAINER: Container, this parameter should indicate which container within the database should be used to store or retrieve the data. This parameter can be compared to the table part of an SQL table reference (DATABASE.TABLE).
  • Paging parameters: Query results may possibly return large amounts of data, this means that a paging mechanism should be set in place:
    • kAPI_PAGE_START: Page start, this parameter indicates the starting page or record.
    • kAPI_PAGE_LIMIT: Page count, this parameter indicates the maximum number of pages or records that the operation should return.
  • Data parameters: A query is formed by a series of sections, each of which is provided with the following parameters:
    • kAPI_QUERY: Query, this parameter is used when retrieving data, it represents the filter or selection query; it must be expressed as one of the query CQuery siblings. This parameter can be compared to the WHERE part of an SQL query.
    • kAPI_SELECT: Fields, this parameter indicates which elements of the selected objects we want returned. This parameter can be compared to the SELECT part of an SQL query.
    • kAPI_SORT: Sort fields, this parameter indicates the sort order, the list of data elements by which the result is to be sorted. This parameter can be compared to the kAPI_SELECT parameter or to the ORDER BY part of an SQL query.
    • kAPI_OBJECT: Object data, this parameter represents the data to be stored in the database, the type should ideally be abstracted from the data store engine. This parameter can be compared to the VALUES or SET part of an SQL query.
    • kAPI_OPTIONS: Options, this parameter represents the options governing data store and retrieve operations. In general it will cover the options when storing data and the actual implementation is the responsibility of derived classes:
      • kAPI_OPT_SAFE: Safe commit option, this is relevant only when committing data. If this option is OFF, it means we want to perform an asynchronous operation: the store operation will occur in the background and the program execution will not wait for it to finish; this also means that the client is responsible for checking whether the operation completed. If the option is ON, the operation is synchronous, which means that the program will wait for the store operation to complete.
      • kAPI_OPT_FSYNC: File sync option, this tag is relevant only when committing data. If the option is ON, it means that the store operation will wait until the data is actually written to disk; which may not necessarily be the case even if the kAPI_OPT_SAFE option was on. When this option is set, it is implied that the kAPI_OPT_SAFE option is also on. If the option is OFF, it means that the data will be synched to disk only when the buffer is flushed.
      • kAPI_OPT_TIMEOUT: Operation timeout, it represents the time in milliseconds beyond which the client will stop waiting for a response and expect a time out status.
      • kAPI_OPT_SINGLE: First element, this option is used by the kAPI_OP_DEL operation: if ON, only the first object satisfying the kAPI_QUERY will be deleted; if OFF all selected elements will be deleted.

The new operations declared in this class are:

  • kAPI_OP_COUNT: This operation requests a count, which is an integer indicating the total number of elements satisfying the provided kAPI_QUERY. This number is not to be confused with the page element kAPI_RESP_PAGE_COUNT described further.
  • kAPI_OP_MATCH: This operation is equivalent to a read query, except that it will try to match one kAPI_QUERY clause at the time and will return a result on the first match.
  • kAPI_OP_GET: This operation is equivalent to a read query, it requests a list of objects satisfying the provided kAPI_QUERY.
  • kAPI_OP_SET: This operation is equivalent to an insert for new objects or an update for existing objects, the operation will replace the object in the data store with the one provided in the kAPI_OBJECT parameter.
  • kAPI_OP_UPDATE: This operation is equivalent to an update operation, this implies that the object must already exist in the data store and that the operation will replace the object in the data store with the one provided in the kAPI_OBJECT parameter.
  • kAPI_OP_INSERT: This operation is equivalent to an insert operation, this implies that the object must not already exist in the data store.
  • kAPI_OP_BATCH_INSERT: This operation is equivalent to the kAPI_OP_INSERT operation, except that we provide here a list of objects to be inserted.
  • kAPI_OP_MODIFY: This operation indicates that we want to modify the contents of an existing object and that the kAPI_OBJECT data represents only the changed elements.
  • kAPI_OP_DEL: This operation indicates that we want to delete the elements matching the provided kAPI_QUERY: the first one only, if the provided kAPI_OPT_SINGLE option is on, or all if off or omitted.

The added functionality implies that a series of additional sections will be returned in the response:

  • kAPI_PAGING: Paging section, this section will return the paging information of the current operation, besides the provided kAPI_PAGE_START and kAPI_PAGE_LIMIT parameters, it will also feature:
    • kAPI_RESP_PAGE_COUNT: This element will hold the actual number of returned objects, this number will be either equal or smaller than the provided kAPI_PAGE_LIMIT parameter.
