Class CTerm

Term object ancestor

A term can be compared to a word of a vocabulary, it is an elementary component that is an abstract concept if considered by itself, but takes a precise meaning if related to other terms, as a sequence of words that constitute a statement.

Terms are uniquely identified by their global identifier, kOFFSET_GID, which is a string code formed by the concatenation of two elements of the object: the namespace, kOFFSET_NAMESPACE, which represents the group to which the term belongs and the local identifier, kOFFSET_LID, which represents the unique identifier of the term within its namespace. These two elements are separated by the kTOKEN_NAMESPACE_SEPARATOR token and the resulting string is set into the object's kOFFSET_GID.

In this class we do not hash the native identifier, kOFFSET_NID.

When the object is committed, CStatusDocument::_IsCommitted(), besides the global and native identifiers, also the kOFFSET_NAMESPACE and kOFFSET_LID offsets will be locked, since they are used to generate the global identifier, kOFFSET_GID.

The object will have its CStatusDocument::_IsInited() status set if the local unique identifier, kOFFSET_LID, is set.

Finally, the class features member accessor methods for the default offsets:

  • CTerm::NS(): This method manages the term's namespace, kOFFSET_NAMESPACE.
  • CTerm::LID(): This method manages the local unique identifier, kOFFSET_LID.

Manage namespace native identifier

Manage namespace native identifier

The namespace native identifier, kOFFSET_NAMESPACE, holds the native identifier of the object that represents the term's namespace.

The method accepts a parameter which represents either the namespace or the requested operation, depending on its value:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • other: Set the value with the provided parameter.

The second parameter is a boolean which if TRUE will return the old value when replacing containers; if FALSE, it will return the currently set value.

Note that when the object has the CStatusDocument::_IsCommitted() status this offset will be locked and an exception will be raised.

Manage local unique identifier

The local unique identifier, kOFFSET_LID, holds a string which represents the object's unique identifier within its namespace, this value is concatenated to the eventual's namespace's global identifier to form the term's global identifier.

Return the object's global unique identifier

Return the object's global unique identifier

Term identifiers are constituted by the concatenation of the namespace and the local unique identifier of the current term separated by a kTOKEN_NAMESPACE_SEPARATOR token.

If the term lacks a namespace, its local identifier will become its global identifier.

Both the namespace and the local identifier will be converted to strings to compute the global identifier.

Handle offset before setting it

The kOFFSET_NAMESPACE and kOFFSET_LID are locked if the object was committed, CStatusDocument::_IsCommitted().
