Class CTerm

Term object ancestor

A term is an object that features the following default properties:

  • Local identifier: The local identifier, CTerm::LID() or the kTAG_LID tag, is a string code that uniquely identifies the object among all those sharing the same namespace. This attribute is required.
  • Global identifier: The global identifier, CPersistentObject::GID() or the kTAG_GID tag, is a string code that uniquely identifies the object among all other objects, regardless of their namespace. This attribute is required and will be generated.
  • Namespace: The namespace, CTerm::NS() or the kTAG_NAMESPACE tag, is a container for a set of term local identifiers allowing disambiguation or homonym local term identifiers. The current term global identifier is generated by concatenating the namespace global identifier to the term's local identifier separated by the kTOKEN_NAMESPACE_SEPARATOR token. This attribute is optional.
  • Label: The label or name, CTerm::Label() or the kTAG_LABEL tag, represents the label, name or short description of the term. This attribute is constituted by an array of elements in which the value of the element represents the label string and the element key is the language code in which the label is expressed in. This attribute is optional.
  • Description: The description or definition, CTerm::Description() or the kTAG_DESCRIPTION tag, represents the description or long label of the term. This attribute is constituted by an array of elements in which the value of the element represents the description string and the element key is the language code in which the description is expressed in. This attribute is optional.
  • Synonyms: The synonym or alias, CTerm::Synonym() or the kTAG_SYNONYMS tag, represents a list of strings that represent alternative names or identifiers for the term. These strings are comparable to variable names and do not have a language scope. This attribute is optional.
  • Term: The term, CTerm::Term() or the kTAG_TERM tag, is a reference to another term that hosts the attributes of the current term. This can be used to redirect synonymous terms to a single instance that holds all the attributes common to the referencing terms. This is used to prevent duplicating attributes for synonym terms. This attribute is constituted by the kOFFSET_NID of the referenced term and is optional.

The native identifier of the term, kOFFSET_NID, is equivalent to the global identifier, in derived classes the native identifier will generally be a hash of the global identifier.

Once the object has been committed, besides the global and native identifiers, also the kTAG_NAMESPACE and kTAG_LID attributes will be locked, since they are used to generate the unique identifier.

The object will have its CStatusDocument::_IsInited() status set if the local identifier, kTAG_LID, is set.

Manage namespace

Manage namespace

The namespace, kTAG_NAMESPACE, holds the prefix of the current term's global identifier, kTAG_GID: the latter is constituted by concatenating the namespace with the local identifier, kTAG_LID, separated by a kTOKEN_NAMESPACE_SEPARATOR token.

The namespace acts as a container for a set of term local identifiers, allowing for homonym local term identifiers.

The method accepts a parameter which represents either the namespace or the requested operation, depending on its value:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • other: Set the value with the provided parameter.

The second parameter is a boolean which if TRUE will return the old value when replacing containers; if FALSE, it will return the currently set value.

Note that when the object has the CStatusDocument::_IsCommitted() status this offset will be locked and an exception will be raised.

Manage term label

Manage term label

The term label, kTAG_LABEL, represents the term's name or short human readable description. It is an optional attribute of the object that holds an array of elements in which the index is represented by the language code and the value is the string.

No two elements may share the same language code.

The method accepts the following parameters:

  • $theLanguage: Language code.
  • $theValue: The label string or the operation, depending on its value:
    • NULL: Return the string corresponding to the provided language.
    • FALSE: Delete the element corresponding to the provided language.
    • other: Any other value represents the label string that will be set or replace the entry for the provided language.
  • $getOld: If TRUE, the method will return the label before it was eventually modified; if FALSE, the method will return the value after eventual modifications.

Manage term synonyms

Manage term synonyms

This method can be used to manage the term's synonyms, kTAG_SYNONYMS, which contains a list of strings that represent alternate codes or names that can be used to identify the term.

This offset collects the list of synonyms in an enumerated set that can be managed with the following parameters:

  • $theValue: Depending on the next parameter, this may either refer to the value to be set or to the index of the element to be retrieved or deleted:
    • NULL: This value indicates that we want to operate on all elements, which means, in practical terms, that we either want to retrieve or delete the full list. If the operation parameter resolves to TRUE, the method will default to retrieving the current list and no new element will be added.
    • array: An array indicates that we want to operate on a list of values and that other parameters may also be provided as lists. Note that ArrayObject instances are not considered here as arrays.
    • other: Any other type represents either the new value to be added or the index to the value to be returned or deleted.
  • $theOperation: This parameter represents the operation to be performed whose scope depends on the value of the previous parameter:
    • NULL: Return the element or full list.
    • FALSE: Delete the element or full list.
    • array: This type is only considered if the $theValue parameter is provided as an array: the method will be called for each element of the $theValue parameter matched with the corresponding element of this parameter, which also means that both both parameters must share the same count.
    • other: Add the $theValue value to the list. If you provided NULL in the previous parameter, the operation will be reset to NULL.
  • $getOld: Determines what the method will return:
    • TRUE: Return the value before it was eventually modified.
    • FALSE: Return the value after it was eventually modified.

Manage term reference

Manage term reference

This method can be used to manage the term reference, kTAG_TERM, which represents the term that hosts all the attributes for the current term.

This attribute is used when defining exact term synonyms: instead of duplicating the attributes in all the synonyms, one can store them in one term and have all the other synonyms point to that term, that way storage and changes reside in a single place.

The method accepts a parameter which represents either the term, or the requested operation, depending on its value:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • other: Set the value with the provided parameter.

The second parameter is a boolean which if TRUE will return the old value when replacing containers; if FALSE, it will return the currently set value.

Return the object's global unique identifier

Return the object's global unique identifier

Term identifiers are constituted by the concatenation of the namespace and the local unique identifier of the current term separated by a kTOKEN_NAMESPACE_SEPARATOR token, this code is generated by the static CTerm::TermCode() method.

If the term lacks a namespace, its local identifier will become its global identifier.

Both the namespace and the local identifier will be converted to strings to compute the global identifier.
