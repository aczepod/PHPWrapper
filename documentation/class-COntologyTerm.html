Class COntologyTerm

Ontology term object ancestor

This class extends its ancestor, CTerm, by adding new rules and reference management.

In this class, the namespace, kOFFSET_NAMESPACE, is a reference to another object derived from this class. This means that the offset will contain the native identifier of the namespace term and that this namespace offset must reside in the same container as the current term.

The class features an offset, kOFFSET_REFS_NAMESPACE, that keeps a reference count of how many times the current term has been referenced as namespace, that is, the number of times this term was used as a namespace by other terms. This offset is updated in the CPersistentDocument::_Postcommit() method.

The class features also two other offsets, kOFFSET_REFS_NODE and kOFFSET_REFS_TAG, which collect respectively the list of node identifiers who reference this term, and the list of tags that reference the term. These offsets are managed by the referenced objects.

The class implements the static method, COntologyTerm::DefaultContainer(), it will use the kCONTAINER_TERM_NAME constant.

The class features member accessor methods for the default offsets:

  • COntologyTerm::NamespaceRefs(): This method returns the term's namespace references, kOFFSET_REFS_NAMESPACE.
  • COntologyTerm::NodeRefs(): This method returns the term's node references, kOFFSET_REFS_NODE.
  • COntologyTerm::TagRefs(): This method returns the term's tag references, kOFFSET_REFS_TAG.
  • COntologyTerm::LoadNamespace(): This method will return the eventual namespace object pointed by kOFFSET_NAMESPACE. kOFFSET_REFS_NAMESPACE.

Load namespace object

Load namespace object

This method will return the current namespace term object: if the namespace is not set, the method will return NULL; if the namespace cannot be found, the method will raise an exception.

The object will also be loaded in a data member that can function as a cache.

The method features two parameters: the first refers to the container in which the namespace is stored, the second is a boolean flag that determines whether the object is to be read, or if the cached copy can be used.

# _Preset( reference & $theOffset, reference & $theValue )

Handle offset before setting it

Handle offset before setting it

In this class we prevent the modification of the three kOFFSET_REFS_NAMESPACE, kOFFSET_REFS_NODE and kOFFSET_REFS_TAG offsets in all cases, since they must be programmatically managed directly by the container and not through the object.

We also handle the namespace offset, if provided as an object we leave it unchanged only if not yet committed, if not, we convert it to its native identifier. We also ensure the provided namespace object to be an instance of this class by asserting CDocument descendants to be of this class; any other type is assumed to be the namespace identifier.

In this class the local identifier, kOFFSET_LID, is a string, in this method we cast the parameter to that type.
