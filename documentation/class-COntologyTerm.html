Class COntologyTerm

Ontology term object ancestor

This class extends its ancestor, CTerm, by implementing a term object in the context of an ontology.

An ontology term represents a concept unrelated to a context. As a vocabulary word changes its meaning depending on its position in a sentence, the same term may be used in different places in the ontology, meaning in different contexts, to represent different things. This class overloads its ancestor to implement the standards and rules that make it into an ontology term.

  • Namespace: The namespace, CTerm::NS() or the inherited kTAG_NAMESPACE tag, in this class is a reference to another instance of this class whose identifier is used as the namespace for the current term's global identifier. The current term's kTAG_GID is generated by concatenating the namespace's global identifier to the term's local identifier separated by the kTOKEN_NAMESPACE_SEPARATOR token. The namespace attribute is constituted by the kTAG_NID of the referenced term and is optional.
  • Namespace references: The namespace references, kTAG_NAMESPACE_REFS tag, represents either the number or the list of instances of this class that reference the current term as a namespace. This attribute is either an integer or an array, depending on the value of the kSWITCH_kTAG_NAMESPACE_REFS flag:
    • 0x2: Keep count of namespace references. This means that the kTAG_NAMESPACE_REFS attribute will be a reference count.
    • 0x3: Keep list of namespace references. This means that the kTAG_NAMESPACE_REFS attribute will be a list of references.
    • 0x0 or other: Don't handle this information. This means that the kTAG_NAMESPACE_REFS attribute will not be handled.
    This attribute is read-only and is managed internally.
  • Node references: The node references, kTAG_NODES tag, represents the references to this object from node objects. This attribute behaves like the kTAG_NAMESPACE_REFS tag, and is managed by the kSWITCH_kTAG_NODES switch.
  • Feature tag references: The feature tag references, kTAG_FEATURES tag, represents references to COntologyTag instances that reference the current term as a kKIND_FEATURE feature. It is either the count or an array of COntologyTag kTAG_NID attributes whose first vertex node in their kTAG_PATH attribute refers to the current term. This attribute behaves like the kTAG_NAMESPACE_REFS tag, and is managed by the kSWITCH_kTAG_FEATURES switch.
  • Method tag references: The method tag references, kTAG_METHODS tag, represents references to COntologyTag instances that reference the current term as a kKIND_METHOD method. It is either the count or an array of COntologyTag kTAG_NID attributes whose vertex nodes in their kTAG_PATH attribute refer to the current term and are not the first or last in the list. This attribute behaves like the kTAG_NAMESPACE_REFS tag, and is managed by the kSWITCH_kTAG_METHODS switch.
  • Scale tag references: The scale tag references, kTAG_SCALES tag, represents references to COntologyTag instances that reference the current term as a kKIND_SCALE scale. It is either the count or an array of COntologyTag kTAG_NID attributes whose last vertex node in their kTAG_PATH attribute refers to the current term. This attribute behaves like the kTAG_NAMESPACE_REFS tag, and is managed by the kSWITCH_kTAG_SCALES switch.

This class prevents updating the full object once it has been inserted for the first time. This behaviour is necessary because terms are referenced by many other objects, so updating a full term object is risky, since it may have been updated elsewhere: for this reason the COntologyTerm::Update() and COntologyTerm::Replace() methods will raise an exception.

The class implements the static method, COntologyTerm::DefaultContainer(), which, given a database, will return the default terms container; it will use the kCONTAINER_TERM_NAME constant as the container name.

# HandleLabel( CConnection $theConnection, mixed $theIdentifier, mixed $theLanguage = NULL, mixed $theValue = NULL )

Modify label

Modify label

This method can be used to add or remove labels from a term, the method will not operate on the object directly, but rather let the container make the modification directly on the database.

The method allows you to add or delete a specific label, it accepts the following parameters:

  • $theIdentifier: This parameter represents the term reference, it can be provided as a term object, as the term native identifier or as the term global identifier. If the term cannot be found, the method will raise an exception.
  • $theLanguage: Language code, NULL refers to the element lacking the language code.
  • $theValue: The label string or the operation, depending on its value:
    • NULL: Return the string corresponding to the provided language.
    • FALSE: Delete the element corresponding to the provided language.
    • other: Any other value represents the label string that will be set or replace the entry for the provided language.

# Resolve( CConnection $theConnection, string $theIdentifier, mixed $theNamespace = NULL, boolean $doThrow = FALSE )

Resolve a term

Resolve a term

This method can be used to locate a term given the attributes that comprise its identifier.

The method accepts the following parameters:

  • $theConnection: This parameter represents the connection from which the terms container must be resolved. If this parameter cannot be correctly determined, the method will raise an exception.
  • $theIdentifier: If the namespace parameter is provided, this parameter should represent the local identifier; if the namespace parameter was omitted, this parameter should represent the native or global identifier.
    • Namespace provided: If the namespace was provided, this parameter is interpreted as the term's local identifier and will be combined with the resolved namespace global identifier to locate the desired term.
    • Namespace not provided: If the namespace was not provided, the method will perform the following queries in order:
      • Native identifier: The method will use the parameter as the native identifier.
      • Global identifier: The method will use the term's COntologyTerm::_id() method to convert the global identifier into a native identifier.
      If the parameter was provided as an instance of COntologyTerm, this method will use its global identifier for the search.
  • $theNamespace: The term namespace:
    • NULL: This indicates that either the term has no namespace, which means that the first parameter must either be the native or global identifier, or that the first parameter is all that is needed to locate the term.
    • COntologyTerm: This type is expected to be the namespace term object:
      • Object is committed: The method will use the namespace's global identifier and concatenate it to the first parameter.
      • Object is not committed: The method will use the namespace's global identifier; if that is missing, the method will either raise an exception, or return NULL, depending on the third parameter.
    • other: Any other type will be interpreted as the namespace's native or global identifier and will be resolved using this same method; the third parameter also applies to this quest.
  • $doThrow: If TRUE, any failure to resolve the term or its namespace, will raise an exception.

The method will return the found term, NULL if not found, or raise an exception if the last parameter is TRUE.

Note: do not provide an array containing the object in the identifier parameter, or you will get unexpected results.

# _Preset( reference & $theOffset, reference & $theValue )

Handle offset before setting it

Handle offset before setting it

In this class we prevent the modification of the kTAG_NAMESPACE_REFS, kTAG_NODES, kTAG_FEATURES, kTAG_METHODS and kTAG_SCALES offsets in all cases, since they must be programmatically managed directly by the container and not through the object.

We also handle the namespace offset, if provided as an object we leave it unchanged only if not yet committed, if not, we convert it to its native identifier. We also ensure the provided namespace object to be an instance of this class by asserting CDocument descendants to be of this class; any other type is assumed to be the namespace identifier.

In this class the local identifier, kTAG_LID, is a string, in this method we cast the parameter to that type.

# _PostcommitRelated( reference & $theConnection, reference & $theModifiers )

Update related objects after committing

Update related objects after committing

In this class we handle referenced namespace terms, kTAG_NAMESPACE: depending on the value of the kSWITCH_kTAG_NAMESPACE_REFS flag we do the following:

  • 0x2: Keep count of namespace references. This means that the kTAG_NAMESPACE_REFS attribute of the term referenced by the kTAG_NAMESPACE attribute will be incremented when the object is inserted, or decremented when deleted.
  • 0x3: Keep list of namespace references. This means that a reference to the current object will be added to the kTAG_NAMESPACE_REFS attribute of the term referenced by kTAG_NAMESPACE attribute of the current object when the latter is inserted, and removed when the object is deleted.
  • 0x0 or other: Don't handle this information. This means that the kTAG_NAMESPACE_REFS attribute will not be handled.
