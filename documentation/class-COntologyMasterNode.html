Class COntologyMasterNode

Ontology master node object

This class extends its COntologyNode parent by enforcing the following rule: There cannot be two nodes that share the same term.

Whenever a new node is instantiated, at commit time the system will check whether another node of the same class, kTAG_CLASS, shares the same term, kTAG_TERM: if that is the case the kTAG_CATEGORY, kTAG_KIND and kTAG_TYPE of the current node will be appended to the corresponding attributes of the referenced node which will become the current node; if that is not the case, the class will behave as its parent and instantiate a new node.

This class also overloads the COntologyMasterNode::RelateTo() method by ensuring that the object of the relationship is an instance of this class, or an COntologyAliasNode which points to an instance of this class.

Relate to node

Relate to node

We overload the inherited method to ensure that both vertices of the relationship do not have node references, kTAG_NODE. If any do have such references, we use the referenced node instead of the current one.

Both the current node and the object node will be committed if not already.

Since this operation involves searching the database, we require the connection which will be used to commit the edge.

# Resolve( CConnection $theConnection, mixed $theIdentifier, boolean $doThrow = FALSE )

Resolve a node

Resolve a node

We overload this method in this class by intercepting all cases in which the $theIdentifier is not an integer, or node reference: since no two nodes of this class can share the same term reference, we add a clause to the search query to exclude all node that have a kTAG_NODE attribute. We do this rather than use the object class, because we assume all nodes that have the kTAG_NODE attribute yo be aliases.

Since terms become also a unique identifier, this method will return either an object or NULL.

# _PrecommitIdentify( reference & $theConnection, reference & $theModifiers )

Determine the identifiers before committing

Determine the identifiers before committing

We overload this method to prevent creating two nodes that refer to the same term. We first check if there is another node that refers to the same term and that does point to another node: if we find one we add the kTAG_CATEGORY, kTAG_KIND and kTAG_TYPE attributes to the found node and make it the current node; if no such node is found, we call the parent method.

In the first case we return the found node identifier, this will stop the commit workflow.
