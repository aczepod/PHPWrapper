Class COntology

Ontology object

This class represents an object whose duty is to provide a high level interface for managing an ontology.

The class features a series of public methods that can be used to instantiate and relate ontology related objects.

One instantiates the object witha CDatabase instance which refers to the database in which the ontology resides.

# NewTerm( string $theIdentifier, mixed $theNamespace = NULL, string $theLabel = NULL, string $theDescription = NULL, string $theLanguage = NULL, array $theAttributes = NULL )

Instantiate a term

Instantiate a term

This method can be used to create a new term, or to retrieve an existing term.

The method will first attempt to locate a term that corresponds to the identifier generated with the provided attributes, if the term is found, it will be returned and the other parameters will be ignored. If the term is not found, a new one will be instantiated and committed before being returned.

The main reason for committing objects immediately is that once an object is saved, we will only issue attributes modifications, we shall never load the whole object and replace it. This is necessary in order to prevent the need of complex locking mechanisms, since ontology elements share many references among each other. For this reason you should first call COntology::ResolveTerm() before calling this method.

The method expects the following parameters:

  • $theIdentifier: This parameter represents the term local identifier.
  • $theNamespace: This parameter represents the term namespace, it must resolve to a committed term or the method will fail. It can be provided in several ways:
    • NULL: This indicates that the term has no namespace.
    • COntologyTerm: This represents the term namespace object.
    • other: Any other type is interpreted as the namespace native or global identifier; if the namespace cannot be found, the method will raise an exception.
  • $theLabel: The term label string, this parameter is optional.
  • $theDescription: The term description string this parameter is optional.
  • $theLanguage: The language code of both the label and the description.
  • $theAttributes: An optional list of additional key/value attributes.

The method will first attempt to locate an existing term. if that fails, it will create a new one, commit it and return it.

# NewNode( mixed $theIdentifier, mixed $theKind = NULL, string $theType = NULL, mixed $theNamespace = NULL, string $theLabel = NULL, string $theDescription = NULL, string $theLanguage = NULL, boolean $doNew = FALSE )

Locate or instantiate a qualified node

Locate or instantiate a qualified node

This method can be used to instantiate a new node, retrieve an existing node by identifier, or retrieve the list of nodes matching a term.

The method expects either a node identifier or a term reference. In the first case the method will return the matching node or NULL. In the second case the method will either return the list of nodes related to the provided term, or a new node related to that term.

This means that when you provide data for a new term, this will be instantiated and committed in all cases.

The method also accepts the node qualifications through the CNode::Kind() and CNode:Type() qualifiers: when resolving nodes, these have to match or a new instance is returned; when creating new nodes, these qualifiers will be set to the provided values.

The method expects the following parameters:

  • $theIdentifier: This parameter represents either the node identifier, or the term reference, depending on the value type:
    • integer: In this case the method assumes that the parameter represents the node identifier: it will attempt to retrieve the node, if it is not found, the method will return NULL. With this option all other parameter are ignored.
    • other: Any other value indicates that the parameter represents a reference to a term, either as a term object or identifier. The term will be resolved: if the $doNew parameter is TRUE, the method will return a new node related to the resolved term; if the $doNew parameter is not TRUE:
      • The term is committed: The method will try to locate all nodes related to that term, if any are found, the method will return the list of nodes; if none are found, the method will return a new node related to that provided term.
      • The term is not committed: The ethod will return a new node related to that term.
    The next four parameters have a meaning only if you are creating a new node with a new term, in all other cases these parameters will be ignored.
  • $theKind: If not NULL, this parameter will serve as a filter to locate matching nodes, or it will be set in new nodes. This parameter represents the node's CNode::Kind(). The parameter can be provided both as an array or as a scalar, in the first case it means that all elements must match.
  • $theType: If not NULL, this parameter will serve as a filter to locate matching nodes, or it will be set in new nodes. This parameter represents the node's CNode::Type(). The parameter is expected to be a scalar.
  • $theNamespace: The term namespace, it can be provided in several ways:
    • NULL: This indicates that the term has no namespace.
    • COntologyTerm: This represents the term namespace object.
    • other: Any other type is interpreted as the namespace native identifier; if the namespace cannot be found, the method will raise an exception.
  • $theLabel: The term label string, this parameter is optional.
  • $theDescription: The term description string this parameter is optional.
  • $theLanguage: The language code of both the label and the description.
  • $doNew: If TRUE, a new node will be instantiated in all cases, that means that the method will not attempt to find a node related to the term. Note that if the first parameter is an integer, this parameter is ignored.

The method will raise an exception if the object is not CStatusDocument::_IsInited() and if the first parameter is NULL.

Add an element to a tag

Add an element to a tag

This method can be used to instantiate and append elements to a COntologyTag, the method will perform specific checks to ensure elements added to the tag are of the correct kind.

The method expects the following parameters:

  • &$theTag: This parameter is a reference to a tag, if the provided parameter is a COntologyTag, it will be used to append elements, if not, it will be instantiated.
  • $theItem: This parameter represents either the item to be appended or the operation:
    • COntologyNode: This type will be interpreted as a node to be added.
    • COntologyTerm: This type will be interpreted as a predicate term to be added.
    • integer: This type will be interpreted as a node identifier.
    • TRUE: This value is interpreted as the command to commit the tag.
    • other: Any other type is interpreted as the predicate term identifier (native or global).

The method makes use of the COntologyTag::PushItem() method, please consult that method to determine what elements can be appended.

Note: provided nodes must be committed, or the method will raise an exception.

When appending elements, the method will return the current number of elements in the path; when committing the tag, the method will return its native identifier.

If any error occurs, the method will raise an exception.

public COntologyTerm
# ResolveTerm( string $theIdentifier, mixed $theNamespace = NULL, boolean $doThrow = FALSE )

Find a term

Find a term

This method can be used to locate a term given the attributes that comprise its identifier.

The method accepts two parameters:

  • $theIdentifier: This parameter can be the term native or global identifier, if the namespace is not provided, or the local identifier if the namespace is provided:
    • Namespace provided: The method will try to resolve the namespace and combine the namespace global identifier with the provided local identifier to locate the term.
    • Namespace not provided: If the namespace was not provided, the method will perform the following queries in order:
      • Native identifier: The method will use the parameter as the native identifier.
      • Global identifier: The method will use the term's COntologyTerm::_id() method to convert the global identifier into a native identifier.
  • $theNamespace: The term namespace:
    • NULL: This indicates that either the term has no namespace, which means that the first parameter must either be the native or global identifier, or that the first parameter is all that is needed to locate the term.
    • COntologyTerm: This type is expected to be the namespace term object:
      • Object is committed: The method will use the namespace's global identifier and concatenate it to the first parameter.
      • Object is not committed: The method will use the namespace's global identifier; if that is missing, the method will either raise an exception, or return NULL, depending on the third parameter.
    • other: Any other type is interpreted as the term native identifier; if the namespace term cannot be found, the method will either raise an exception, or return NULL, depending on the third parameter.
  • $doThrow: If TRUE, any failure to resolve the term or its namespace, will raise an exception.

The method will return the found term, NULL if not found, or raise an exception if the third parameter is TRUE.

This class takes advantage of the static method COntologyTerm::Resolve().

# ResolveEdge( mixed $theSubject = NULL, mixed $thePredicate = NULL, mixed $theObject = NULL, boolean $doThrow = FALSE )

Find an edge

Find an edge

This method can be used to retrieve an existing edge by providing the elements that constitute it.

The method expects the following parameters:

  • $theSubject: This parameter is interpreted as the edge subject vertex node reference:
    • COntologyNode: It will be interpreted as the node object.
    • integer: It will be interpreted as the node native identifier.
    • other: The method will assume the value to be a term reference: in this case the method will use the list of nodes that reference the eventual referenced term.
  • $thePredicate: This parameter is interpreted as the edge predicate term reference:
    • COntologyTerm: It will be interpreted as the term object.
    • other: The method will attempt to resolve the term by testing in turn the value as its global and native identifier.
  • $theObject: This parameter is interpreted as the edge object vertex node reference:
    • COntologyNode: It will be interpreted as the node object.
    • integer: It will be interpreted as the node native identifier.
    • other: The method will assume the value to be a term reference: in this case the method will use the list of nodes that reference the eventual referenced term.
  • $doThrow: If this parameter is TRUE and no edge is found, the method will raise an exception.

If any of the provided parameters fail to resolve, the method will assume the edge cannot be found.

The method will return an array of COntologyEdge instances, or NULL if no edge was found.

The method will raise an exception if the object is not CStatusDocument::_IsInited().

Create an edge

Create an edge

This method can be used to instantiate an edge by providing the subject and object vertex nodes and the edge predicate term. The method makes use of the node's CNode::RelateTo() method to perform the actual operation. This method will ensure that the subject node is committed before calling its method and eventually create a new predicate term from its provided attributes.

The method expects the vertex nodes to be provided as node references, the predicate term can be created by this method by providing its attributes.

  • $theSubject: Subject node reference.
  • $theObject: Object node reference.
  • $thePredicate: This parameter represents the predicate term reference, object or local identifier.
  • $theNamespace: The predicate namespace, it can be provided in several ways:
    • NULL: This indicates that the term has no namespace.
    • COntologyTerm: This represents the term namespace object.
    • other: Any other type is interpreted as the namespace native identifier; if the namespace cannot be found, the method will raise an exception.
  • $theLabel: The term label string, this parameter is optional.
  • $theDescription: The term description string this parameter is optional.
  • $theLanguage: The language code of both the label and the description.

The method will return a committed instance of the COntologyEdge class, if any error occurs, the method will raise an exception.

# ExportTerm( reference & $theCollection, mixed $theTerm, array $theAttributes = NULL, boolean $doTags = TRUE )

Export a term

Export a term

The main duty of this method is to normalise and merge the provided term's attributes and place this information in the related provided collection container.

The method expects the following parameters:

  • &$theCollection: This parameter is a reference to an array that will receive the object attributes, this collection is divided in four elements:
    • kOFFSET_EXPORT_PREDICATE: This element is an array that holds the list of predicate terms, the array keys will be the term's kTAG_GID and the value will be the attributes of the term. The contents of this element are fed by the COntology::_ExportTerm() protected method.

# _ExportNode( integer $theNode )

Export a node

Export a node

The main duty of this method is to resolve the provided node reference and return a single object that merges all its term and node attributes.

The method will return an array containing the merged attributes of the node and the referenced term. By default, in case of conflict, the node attributes will overwrite the term attributes, in all cases, however, the kTAG_LID and kTAG_GID will be taken from the term.

We omit the kTAG_CLASS, kTAG_TERM (the term's attributes are merged with the node's attributes), kTAG_REFS_TAG and kTAG_REFS_EDGE.
