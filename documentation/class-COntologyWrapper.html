Class COntologyWrapper

Ontology wrapper.

This class overloads its ancestor to implement the framework for providing services that return data from an ontology. The class does not feature a concrete data store engine, so it only concentrates in providing the framework for derived classes that implement concrete data store instances, for this reason this class is abstract.

This class implements high level services, that is, it will return data that is an aggregation of various containers.

These new functionalities require an additional set of parameters:

  • Category parameters: These parameters refer to categories used to select specific elements:
    • kAPI_KIND: Kind, this parameter generally refers to the node kind, it is an array of elements that the selected nodes must match.
  • Operations: This class adds the following operations:
    • kAPI_OP_GetNodesByKind: Get nodes by kind, this operation will return a list of nodes that match the provided kind enumerations in the kAPI_KIND parameter. The operation expects the following parameters:
      • kAPI_FORMAT: Format, this parameter is required, since other non scalar parameters must be encoded.
      • kAPI_DATABASE: Database, database to which the container belongs.
      • kAPI_KIND: Kinds list (optional), the selection criteria.
      • kAPI_SELECT: Select (optional), the list of fields to be returned.
      • kAPI_PAGE_LIMIT:This parameter is required or enforced, it represents the maximum number of elements that the query should return, the default value is kDEFAULT_LIMIT.
