Class COntologyWrapper

Ontology wrapper

This class overloads its ancestor to implement the framework for providing services that return data from an ontology. The class concentrates on high level services that return aggregated elements taken from the term, node, edge and tag collections.

The response parameter will always contain the following elements:

  • kAPI_COLLECTION_ID: This offset tags the element that holds the list of identifiers of the requested items.
  • kAPI_COLLECTION_TERM: This offset tags the element that holds the list of referenced predicate items
  • kAPI_COLLECTION_NODE: This offset tags the element that holds the list of referenced vertex items
  • kAPI_COLLECTION_EDGE: This offset tags the element that holds the list of referenced edge items
  • kAPI_COLLECTION_TAG: This offset tags the element that holds the list of referenced tag items

# _Handle_GetVertex( )

Handle kAPI_OP_GetVertex request.

Handle kAPI_OP_GetVertex request.

This method will handle the kAPI_OP_GetVertex operation, which returns either the list of vertex satisfying the provided query, or the related vertices of the vertex selected by the provided query.

If the kAPI_RELATION parameter is provided, it is assumed that you want the vertices related to the first vertex matched by the provided query, in that case the service will return a list of vertices that either point to the matched vertex, pointed to by the reference vertex, or both.

If the latter parameter is provided, the service may optionally receive the kAPI_SUBQUERY parameter which represents a filter or query that will be applied to the related elements (not to the result of the kAPI_QUERY filter).

If the kAPI_RELATION parameter is not provided, the service will return the vertices that match the provided query.

If the query is omitted, the service will use the provided or default nodes container.

# _ExportTerm( reference & $theCollection, mixed $theTerm, boolean $doTags = TRUE )

Export a predicate

Export a predicate

The main duty of this method is to normalise and merge the provided term's attributes and place this information in the related provided collection container.

The method expects the following parameters:

  • &$theCollection: This parameter is a reference to an array that will receive the object attributes, the array holds four lists that collect similar objects, such as nodes, terms, edges and tags:
    • kAPI_COLLECTION_ID: This element is an array that holds the identifiers list of the requested elements. This list holds the reference to the object which is contained by the other elements of the collection, this list refers to the elements requested by the service.
    • kAPI_COLLECTION_TERM: This element is an array that holds the list of predicate terms referenced by all other objects in the collection. The array keys will be the term's kTAG_GID and the value will be the attributes of the term. The contents of this element are fed by the COntologyWrapper::_BuildTerm() protected method and the elements are provided by this method.
    • kAPI_COLLECTION_NODE: This element is an array that holds a list of node vertex. A vertex is the combination of the node attributes merged with the referenced term attributes. The items of this list are indexed by the node kTAG_NID and eventual term attributes are overwritten by matching node attributes. The contents of this element are fed by the COntologyWrapper::_BuildNode() protected method.
    • kAPI_COLLECTION_EDGE: This element is an array that holds a list of edges, the array keys will be the edge's kTAG_GID and the value will be the edge's attributes. The contents of this element are fed by the _BuildEdge() protected method.
    • kAPI_COLLECTION_TAG: This element is an array that holds a list of tags, the array indexes will be the tag kTAG_NID and the array values will be the tag kTAG_PATH and kTAG_GID, where the kTAG_PATH predicate references will be set to the term kTAG_GID.
  • $theTerm: This parameter represents the vertex node full object; this means that all attributes should be present, since this method will take care of removing unwanted attributes. The parameter may represent a list of node objects or references, a reference or an object:
    • array: Each element of the list will be fed to this method recursively.
    • COntologyNode: The object will be used as is.
    • other: Any other type will be interpreted as a node reference.
  • $theAttributes: This optional parameter can be used to limit the returned attributes to the list provided in this array.
  • $doTags: If this flag is TRUE, the method will load the tags; if FALSE no tags will be loaded.

The method will generate an array containing the merged attributes of the node and the referenced term, this array will be set in the kAPI_COLLECTION_NODE of the &$theCollection parameter with as index the node kTAG_NID that will ve added to the kAPI_COLLECTION_ID element of the collection.

If a matching vertex already exists in the &$theCollection parameter, the method will do nothing.

For more information please consult the COntologyWrapper::_BuildNode() method reference, note that this method will remove the kTAG_NID attribute from the node, it will only use this information to index the node.

The method will raise an exception if any element cannot be resolved.

# _ExportEdge( reference & $theCollection, mixed $theEdge, boolean $doTags = TRUE )

Export an edge

Export an edge

The main duty of this method is to normalise the edge's attributes and store the referenced vertex and predicates in the relative elements of the provided collection.

The method expects the following parameters:

  • &$theCollection: This parameter is a reference to an array that will receive the object attributes, the array holds four lists that collect similar objects, such as nodes, terms, edges and tags:
    • kAPI_COLLECTION_ID: This element is an array that holds the identifiers list of the requested elements. This element should be filled by the caller, since its content depend on what operation is going on.
    • kAPI_COLLECTION_TERM: This element is an array that holds the list of predicate terms referenced by all other objects in the collection. The array keys will be the term's kTAG_GID and the value will be the attributes of the term. The contents of this element are fed with the COntologyWrapper::_BuildTerm() protected method to which the edge predicates will be provided.
    • kAPI_COLLECTION_NODE: This element is an array that holds a list of node vertex. A vertex is the combination of the node attributes merged with the referenced term attributes. The items of this list are indexed by the node kTAG_NID and eventual term attributes are overwritten by matching node attributes. The contents of this element are fed by the COntologyWrapper::_BuildNode() protected method to which both the object and subject vertices will be provided.
    • kAPI_COLLECTION_EDGE: This element is an array that holds a list of edges, the array keys will be the edge's kTAG_GID and the value will be the edge's attributes. The contents of this element are fed by the _BuildEdge() protected method.
    • kAPI_COLLECTION_TAG: This element is an array that holds a list of tags, the array indexes will be the tag kTAG_NID and the array values will be the tag kTAG_PATH and kTAG_GID, where the kTAG_PATH predicate references will be set to the term kTAG_GID.
  • $theEdge: This parameter represents the edge object or a list of edges:
    • array: A list of edges.
    • COntologyEdge: The edge will be used as-is.

The method will omit the kTAG_UID and kTAG_CLASS attributes, the kTAG_PREDICATE attribute will be set to the term's kTAG_GID attribute and the two vertex references will be left untouched. The resulting array will be set into the kOFFSET_EXPORT_EDGE element of the provided collection.

The method will raise an exception if any element cannot be resolved.

# _ExportTag( reference & $theCollection, mixed $theTag )

Export a tag

Export a tag

The main duty of this method is to normalise the tag's attributes and store the referenced predicates and vertices in the relative elements of the provided collection.

The method expects the following parameters:

  • &$theCollection: This parameter is a reference to an array that will receive the object attributes, the array holds four lists that collect similar objects, such as nodes, terms, edges and tags:
    • kAPI_COLLECTION_ID: This element is an array that holds the identifiers list of the requested elements. This element is ignored by this method.
    • kAPI_COLLECTION_TERM: This element is an array that holds the list of predicate terms referenced by all other objects in the collection. The array keys will be the term's kTAG_GID and the value will be the attributes of the term. The contents of this element are fed by the COntologyWrapper::_BuildTerm() protected method.
    • kAPI_COLLECTION_NODE: This element is an array that holds a list of node vertex. A vertex is the combination of the node attributes merged with the referenced term attributes. The items of this list are indexed by the node kTAG_NID and eventual term attributes are overwritten by matching node attributes. The contents of this element are fed by the COntologyWrapper::_BuildNode() protected method.
    • kAPI_COLLECTION_EDGE: This element is an array that holds a list of edges, the array keys will be the edge's kTAG_GID and the value will be the edge's attributes. The contents of this element are fed by the _BuildEdge() protected method.
    • kAPI_COLLECTION_TAG: This element is an array that holds a list of tags, the array indexes will be the tag kTAG_NID and the array values will be the tag kTAG_PATH and kTAG_GID, where the kTAG_PATH predicate references will be set to the term kTAG_GID. This method will fill this element.
  • $theTag: This parameter represents the tag identifier, object or a list of tag identifiers:
    • array: A list of tag or node identifiers.
    • COntologyTag: The tag will be used as-is.
    • other: Any other type will be interpreted as a tag reference and resolved with ResolveTag().
  • $theAttributes: This optional parameter can be used to limit the returned attributes to the list provided in this array.

The method will generate an array containing the merged attributes of the tag and the exported attributes of the referenced nodes, generated by the COntologyWrapper::_ExportNode() protected method. The method will exclude the kTAG_NID, kTAG_UID, and kTAG_CLASS from the tag, but append all others to the merged node and term attributes. The predicate elements of the KTAG_PATH attribute will be set to the referenced term's kTAG_GID. The resulting array will be set in the kOFFSET_EXPORT_TAG element of the provided collection using the tag's kTAG_NID attribute as index.

The method will raise an exception if any element cannot be resolved.

# _BuildTerm( mixed $theTerm )

Build a term

Build a term

The main duty of this method is to resolve the provided term reference and return a single object that holds a merged selection of attributes.

By default we include the kTAG_LID, kTAG_GID and the kTAG_NAMESPACE (resolved into the referenced kTAG_GID) from the current term.

We omit the kTAG_NID, kTAG_CLASS, kTERM_TERM, kTERM_NAMESPACE_REFS and kTERM_NODES.

All other attributes will either be included from the current term, or, if the term is related to another term through its kTERM_TERM attribute, from that term.

The method will raise an exception if any element cannot be resolved.

# _BuildNode( mixed $theNode )

Build a vertex

Build a vertex

The main duty of this method is to resolve the provided node's references and return a single object that merges all its term and node attributes.

Since we use a document database to store the graph vertices index, it is better to store referenced data in the vertex object, so that it is easier to search.

In this class we therefore return the object as-is, in derived classes we can use this method for other purposes.

Select language elements.

This method can be used to restrict the language string attributes, those of type kTYPE_LSTRING, to the list of provided languages.

The method will operate exclusively on the current object's response, kAPI_RESPONSE, property: the method will iterate the term and node sections of the response and consider only those attributes od type kTYPE_LSTRING, if at least one language matches, the method will select only the matching languages; if no language matches, the method will leave the attribute untouched.

# _NormaliseTermReferences( reference & $theQuery )

Filter namespace references.

Filter namespace references.

The duty of this method is to convert term references from kTAG_GID values to kTAG_NID values.

The method will traverse the query structure and intercept the following kOFFSET_QUERY_SUBJECT instances:

  • kTAG_NAMESPACE: The namespace reference of the term.
  • kTAG_TERM: The term reference in the term or node.
  • kTAG_PREDICATE: The predicate reference in the edge.
  • kTAG_UID: Unique identifiers.

Any of these clauses with kOFFSET_QUERY_TYPE kTYPE_STRING will be converted to term native identifiers using the static COntologyTerm::_id() method.

Note: here we set the data type to a binary string, if you decide to change the native identifier data type, you will have to update this method.
