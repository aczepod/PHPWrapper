Class COntologyWrapper

Ontology wrapper.

This class overloads its ancestor to implement the framework for providing services that return data from an ontology. The class concentrates on high level services that return aggregated elements taken from the term, node, edge and tag collections.

The response parameter will always contain the following elements:

  • kAPI_COLLECTION_ID: This offset tags the element that holds the list of identifiers of the requested items.
  • kAPI_COLLECTION_PREDICATE: This offset tags the element that holds the list of referenced predicate items
  • kAPI_COLLECTION_VERTEX: This offset tags the element that holds the list of referenced vertex items
  • kAPI_COLLECTION_EDGE: This offset tags the element that holds the list of referenced edge items
  • kAPI_COLLECTION_TAG: This offset tags the element that holds the list of referenced tag items

# _ExportVertex( reference & $theCollection, mixed $theNode, boolean $doTags = TRUE )

Export a vertex

Export a vertex

The main duty of this method is to provide a common display format for the elements of the graph, given a node identifier or identifiers list, this method will resolve all its references and return a single object that merges all its term and node attributes.

The method expects the following parameters:

  • &$theCollection: This parameter is a reference to an array that will receive the object attributes, the array holds four lists that collect similar objects, such as nodes, terms, edges and tags:
    • kAPI_COLLECTION_ID: This element is an array that holds the identifiers list of the requested elements. This list holds the reference to the object which is contained by the other elements of the collection, this list refers to the elements requested by the service.
    • kAPI_COLLECTION_PREDICATE: This element is an array that holds the list of predicate terms referenced by all other objects in the collection. The array keys will be the term's kTAG_GID and the value will be the attributes of the term. The contents of this element are fed by the COntologyWrapper::_BuildTerm() protected method.
    • kAPI_COLLECTION_VERTEX: This element is an array that holds a list of node vertex. A vertex is the combination of the node attributes merged with the referenced term attributes. The items of this list are indexed by the node kOFFSET_NID and eventual term attributes are overwritten by matching node attributes. The contents of this element are fed by the COntologyWrapper::_BuildVertex() protected method.
    • kAPI_COLLECTION_EDGE: This element is an array that holds a list of edges, the array keys will be the edge's kTAG_GID and the value will be the edge's attributes. The contents of this element are fed by the _BuildEdge() protected method.
    • kAPI_COLLECTION_TAG: This element is an array that holds a list of tags, the array indexes will be the tag kOFFSET_NID and the array values will be the tag kTAG_TAG_PATH and kTAG_GID, where the kTAG_TAG_PATH predicate references will be set to the term kTAG_GID.
  • $theNode: This parameter represents the vertex node full object; this means that all attributes should be present, since this method will take care of removing unwanted attributes. The parameter may represent a list of node objects or references, a reference or an object:
    • array: Each element of the list will be fed to this method recursively.
    • COntologyNode: The object will be used as is.
    • other: Any other type will be interpreted as a node reference.
  • $theAttributes: This optional parameter can be used to limit the returned attributes to the list provided in this array.
  • $doTags: If this flag is TRUE, the method will load the tags; if FALSE no tags will be loaded.

The method will generate an array containing the merged attributes of the node and the referenced term, this array will be set in the kAPI_COLLECTION_VERTEX of the &$theCollection parameter with as index the node kOFFSET_NID that will ve added to the kAPI_COLLECTION_ID element of the collection.

If a matching vertex already exists in the &$theCollection parameter, the method will do nothing.

For more information please consult the COntologyWrapper::_BuildVertex() method reference, note that this method will remove the kOFFSET_NID attribute from the node, it will only use this information to index the node.

The method will raise an exception if any element cannot be resolved.

# _ExportEdge( reference & $theCollection, mixed $theEdge )

Export an edge

Export an edge

The main duty of this method is to normalise the edge's attributes and store the referenced vertex and predicates in the relative elements of the provided collection.

The method expects the following parameters:

  • &$theCollection: This parameter is a reference to an array that will receive the object attributes, the array holds four lists that collect similar objects, such as nodes, terms, edges and tags:
    • kAPI_COLLECTION_ID: This element is an array that holds the identifiers list of the requested elements. This element should be filled by the caller, since its content depend on what operation is going on.
    • kAPI_COLLECTION_PREDICATE: This element is an array that holds the list of predicate terms referenced by all other objects in the collection. The array keys will be the term's kTAG_GID and the value will be the attributes of the term. The contents of this element are fed with the COntologyWrapper::_BuildTerm() protected method to which the edge predicates will be provided.
    • kAPI_COLLECTION_VERTEX: This element is an array that holds a list of node vertex. A vertex is the combination of the node attributes merged with the referenced term attributes. The items of this list are indexed by the node kOFFSET_NID and eventual term attributes are overwritten by matching node attributes. The contents of this element are fed by the COntologyWrapper::_BuildVertex() protected method to which both the object and subject vertices will be provided.
    • kAPI_COLLECTION_EDGE: This element is an array that holds a list of edges, the array keys will be the edge's kTAG_GID and the value will be the edge's attributes. The contents of this element are fed by the _BuildEdge() protected method.
    • kAPI_COLLECTION_TAG: This element is an array that holds a list of tags, the array indexes will be the tag kOFFSET_NID and the array values will be the tag kTAG_TAG_PATH and kTAG_GID, where the kTAG_TAG_PATH predicate references will be set to the term kTAG_GID.
  • $theEdge: This parameter represents the edge object or a list of edges:
    • array: A list of edges.
    • COntologyEdge: The edge will be used as-is.

The method will omit the kTAG_UID and kTAG_CLASS attributes, the kTAG_PREDICATE attribute will be set to the term's kTAG_GID attribute and the two vertex references will be left untouched. The resulting array will be set into the kOFFSET_EXPORT_EDGE element of the provided collection.

The method will raise an exception if any element cannot be resolved.

# _ExportTag( reference & $theCollection, mixed $theTag )

Export a tag

Export a tag

The main duty of this method is to normalise the tag's attributes and store the referenced predicates and vertices in the relative elements of the provided collection.

The method expects the following parameters:

  • &$theCollection: This parameter is a reference to an array that will receive the object attributes, the array holds four lists that collect similar objects, such as nodes, terms, edges and tags:
    • kAPI_COLLECTION_ID: This element is an array that holds the identifiers list of the requested elements. This element is ignored by this method.
    • kAPI_COLLECTION_PREDICATE: This element is an array that holds the list of predicate terms referenced by all other objects in the collection. The array keys will be the term's kTAG_GID and the value will be the attributes of the term. The contents of this element are fed by the COntologyWrapper::_BuildTerm() protected method.
    • kAPI_COLLECTION_VERTEX: This element is an array that holds a list of node vertex. A vertex is the combination of the node attributes merged with the referenced term attributes. The items of this list are indexed by the node kOFFSET_NID and eventual term attributes are overwritten by matching node attributes. The contents of this element are fed by the COntologyWrapper::_BuildVertex() protected method.
    • kAPI_COLLECTION_EDGE: This element is an array that holds a list of edges, the array keys will be the edge's kTAG_GID and the value will be the edge's attributes. The contents of this element are fed by the _BuildEdge() protected method.
    • kAPI_COLLECTION_TAG: This element is an array that holds a list of tags, the array indexes will be the tag kOFFSET_NID and the array values will be the tag kTAG_TAG_PATH and kTAG_GID, where the kTAG_TAG_PATH predicate references will be set to the term kTAG_GID. This method will fill this element.
  • $theTag: This parameter represents the tag identifier, object or a list of tag identifiers:
    • array: A list of tag or node identifiers.
    • COntologyTag: The tag will be used as-is.
    • other: Any other type will be interpreted as a tag reference and resolved with ResolveTag().
  • $theAttributes: This optional parameter can be used to limit the returned attributes to the list provided in this array.

The method will generate an array containing the merged attributes of the tag and the exported attributes of the referenced nodes, generated by the _ExportNode() protected method. The method will exclude the kOFFSET_NID, kTAG_UID, kTAG_CLASS and kTAG_VERTEX_TERMS from the tag, but append all others to the merged node and term attributes. The predicate elements of the KTAG_PATH attribute will be set to the referenced term's kTAG_GID. The resulting array will be set in the kOFFSET_EXPORT_TAG element of the provided collection using the tag's kOFFSET_NID attribute as index.

The method will raise an exception if any element cannot be resolved.

# _BuildVertex( mixed $theNode )

Build a vertex

Build a vertex

The main duty of this method is to resolve the provided node's references and return a single object that merges all its term and node attributes.

The method will return an array containing the merged attributes of the node and the referenced term. By default, in case of conflict, the node attributes will overwrite the term attributes; in all cases, however, the kTAG_LID and kTAG_GID will be taken from the term.

We omit the kTAG_CLASS, kTAG_TERM (the term's attributes are merged with the node's attributes), kTAG_REFS_TAG and kTAG_REFS_EDGE.

All other attributes will either be included from the referenced term, or will be taken by the current node; note that the node attributes will overwrite the term attributes.

The method will raise an exception if any element cannot be resolved.

# _BuildTerm( mixed $theTerm )

Build a term

Build a term

The main duty of this method is to resolve the provided term reference and return a single object that holds a merged selection of attributes.

By default we include the kTAG_LID, kTAG_GID and the kTAG_NAMESPACE (resolved into the referenced kTAG_GID) from the current term.

We omit the kOFFSET_NID, kTAG_CLASS, kOFFSET_TERM, kOFFSET_REFS_NAMESPACE, kOFFSET_REFS_NODE and kOFFSET_REFS_TAG.

All other attributes will either be included from the current term, or, if the term is related to another term through its kOFFSET_TERM attribute, from that term.

The method will raise an exception if any element cannot be resolved.
