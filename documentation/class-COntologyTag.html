Class COntologyTag

Ontology tag object ancestor

This class extends its ancestor, CTag, by asserting the type of objects passed to the path, kOFFSET_PATH, and by using the native identifiers, kOFFSET_NID, of the elements of the path in the COntologyTag::_index() method.

Odd elements of the path must refer to vertex elements of the graph, in this case COntologyNode objects; even elements of the path must refer to predicates of the graph, in this case COntologyTerm objects.

The elements of the path offset, kOFFSET_PATH, will be the native identifier of the path elements, while the global identifier, kOFFSET_GID, will receive the global identifiers of the path element related COntologyTerm instances. This last behaviour means that once a tag is saved, related terms, nodes and edges should not be deleted.

The class adds an offset, kOFFSET_UID, which stores the hashed version of the global identifier and can be used to identify duplicate objects.

Elements of the path can be provided as objects and those that are not yet committed will be saved when the current tag is saved.

Another action tags perform over nodes is the setting of the node CNode::Kind(): the first element of the path is a trait node, kNODE_KIND_TRAIT, the last element of the path is a scale node, kNODE_KIND_SCALE, and the eventual middle elements are method nodes, kNODE_KIND_METH. Once the tag is committed, the node's kind will be updated accordingly.

When adding elements to the tag's path you must consistently follow these rules:

  • The first vertex of the path must represent the descriptor or main category of the tag, it will be qualified as kNODE_KIND_TRAIT.
  • The last vertex of the path must represent the data type of the tag and must have a COntologyNode::Type(), it will be qualified as kNODE_KIND_SCALE.
  • The eventual vertices between the first and the last one should represent variations or subclasses of the first vertex and will be marked as kNODE_KIND_METH.

Because of this rule, it is not guaranteed that tag paths are in sync with graph paths, the predicate term glue is used as a discriminant to differentiate paths holding the same vertices.

The other effect of this is that it is not easy to match the path of a tag with the original path in the graph that was used to constitute the tag, the only thing one can do is ensure tags are constituted from elements following the ontology graph paths.

This class prevents updating the full object once it has been inserted for the first time. This behaviour is necessary because tags are referenced by many other objects, so updating a full tag object is risky, since it may have been updated elsewhere: for this reason the COntologyTag::Update() and COntologyTag::Replace() methods will raise an exception.

Objects of this class use a sequence number as their native identifier tagged as kSEQUENCE_KEY_TAG and feature a default container name kCONTAINER_TAG_NAME.

# Resolve( CConnection $theConnection, mixed $theIdentifier, boolean $doThrow = FALSE )

Resolve a tag

Resolve a tag

This method can be used to retrieve an existing tag by identifier, or retrieve all tags matching a term.

The method accepts the following parameters:

  • $theConnection: This parameter represents the connection from which the tags container must be resolved. If this parameter cannot be correctly determined, the method will raise an exception.
  • $theIdentifier: This parameter represents either the tag identifier or the term reference, depending on its type:
    • integer: In this case the method assumes that the parameter represents the tag identifier: it will attempt to retrieve the tag, if it is not found, the method will return NULL.
    • COntologyTerm: In this case the method locate all tags that refer to the provided term. If the term is not CStatusDocument::_IsCommitted(), the method will return NULL.
    • other: Any other type will be interpreted either the term's native identifier, or as the term's global identifier: the method will return all tags that refer to that term.
  • $doThrow: If TRUE, any failure to resolve the tag will raise an exception.

The method will return the found tag(s), NULL if not found, or raise an exception if the last parameter is TRUE.

# _index( CConnection $theConnection, bitfield $theModifiers )

Return the object's global unique identifier

Return the object's global unique identifier

We override the parent method to handle the referenced objects: in this class the global identifier is the concatenation of all the referenced terms of the tag path, this method will take care of instantiating the necessary objects and building the global identifier.

This method is called by the COntologyTag::_PrecommitIdentify() and expects all elements of the path to be object native identifiers, so the method will resolve each element without checking if it is in the form of an object.

This method is also responsible for loading the kOFFSET_TERM_LIST offset: it will contain all the term native identifiers referred to by the tag's path vertex elements.

Determine the identifiers before committing

Determine the identifiers before committing

This method will use the result of the COntologyTag::_index() method to set the global identifier, kOFFSET_GID; the same value will be hashed and constitute the unique identifier, kOFFSET_UID, while the native identifier, kOFFSET_NID, will be set by the sequence number kSEQUENCE_KEY_TAG.

The parent method will then be called, which will ignore the global and native identifiers, since they will have been set here.

# _PostcommitRelated( reference & $theConnection, reference & $theModifiers )

Update related objects after committing

Update related objects after committing

The current tag will update related vertex object, COntologyNode instances and their relative COntologyTerm instances, both with the kOFFSET_REFS_TAG offset.

When inserting a new tag, we add the tag reference, when deleting the tag we remove it.

The method will also set the node CNode::Kind()s to kNODE_KIND_TRAIT for the first element of the path, to kNODE_KIND_SCALE for the last element and to kNODE_KIND_METH for the eventual in-between node elements.

We only handle vertex elements, we do not refer predicate terms.
