Class CPortalWrapper

Portal wrapper

This class overloads its ancestor to implement a series of general purpose frameworks.

The class adds the following set of parameters:

  • Request parameters: These parameters refer to the parameters that the service expects:
    • kAPI_CREDENTIALS: Credentials, this offset tag will be used to provide login or operation credentials, the parameter is expected to be an array whose structure depends on the operation.
  • Operations: This class adds the following operations:
    • kAPI_OP_Login: Login, this operation will return the user record corresponding to the provided kAPI_CREDENTIALS credentials parameter. The operation expects the following parameters:
      • kAPI_FORMAT: This parameter is required to indicate how to encode the response.
      • kAPI_DATABASE: This parameter is required to indicate the working database.
      • kAPI_CONTAINER: This optional parameter is used to indicate the working container in case of a non-standard configuration.
      • kAPI_CREDENTIALS: This parameter is required for validating the user:
        • kAPI_CREDENTIALS_CODE: User code.
        • kAPI_CREDENTIALS_PASS: User password.
        Note that these parameters may grow in derived classes, this represents the bare minimum.
    • kAPI_OP_NewUser: New user, this operation will create a new user, it expects a set of attributes representing the new user's properties.
      • kAPI_FORMAT: This parameter is required to indicate how to encode the response.
      • kAPI_DATABASE: This parameter is required to indicate the working database.
      • kAPI_OBJECT: This parameter should contain the attributes of the new user.

# _Handle_Login( )

Handle kAPI_OP_Login request.

Handle kAPI_OP_Login request.

This method will handle the kAPI_OP_Login operation, which returns the user record upon successful login.

The method will use the kAPI_CREDENTIALS parameter kAPI_CREDENTIALS_CODE element as the user ID and the kAPI_CREDENTIALS_PASS element as the password.

If the the operation succeeds, the method will return the found user in the response.
