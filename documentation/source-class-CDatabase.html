 29: /**
 30:  * <h4>Database object ancestor</h4>
 31:  *
 32:  * This <i>abstract</i> class is the ancestor of all database classes in this library, it
 33:  * implements the interface and workflow that all concrete derived classes should implement.
 34:  *
 35:  * A database object holds a data member that represents a native connection to a database
 36:  * server, database objects have the ability to generate other connection derived objects
 37:  * that have specific functionality, such as containers.
 38:  *
 39:  * The object is instantiated by providing the native database connection which is stored in
 40:  * a data member of the object, the array part of the object can be used to store specific
 41:  * attributes of the native database connection.
 42:  *
 43:  * The class features an offset, {@link kOFFSET_PARENT}, that can be used to store the
 44:  * parent object, in this case the server object. This member gets set when the object is
 45:  * instantiated by a server object.
 46:  *
 47:  * The abstract class must be overloaded by concrete classes that implement a native
 48:  * database object.
 49:  *
 50:  *  @package    MyWrapper
 51:  *  @subpackage Framework
