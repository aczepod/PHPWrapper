 36: /**
 37:  * <h3>Persistent objects data store ancestor</h3>
 38:  *
 39:  * This <i>abstract</i> class is the ancestor of all container classes in this library, it
 40:  * implements the interface and workflow that all concrete derived classes should implement.
 41:  *
 42:  * The public interface declares the following main operations:
 43:  *
 44:  * <ul>
 45:  *  <li>{@link SetObject()}: This method will <i>insert</i>, <i>update</i>, <i>modify</i> or
 46:  *      <i>delete</i> objects from the current container. This method operates one object at
 47:  *      the time and it is mainly used by persistent objects for persisting in containers.
 48:  *  <li><i>{@link GetObject()}</i>: This method will <i>load</i> an object from the current
 49:  *      container.
 50:  * </ul>
 51:  *
 52:  * The class does not implement a concrete data store, derived classes must implement
 53:  * specific storage actions.
 54:  *
 55:  * The class inherits from {@link CStatusDocument} in order to manage its own status and the
 56:  * one of the provided persistent objects.

205:     /**
206:      * <h4>Manage an object from the container</h4>
207:      *
208:      * This method can be used to manage a single object in the container, it can
209:      * <i>insert</i>, <i>update</i>, <i>replace</i>, <i>modify</i>, <i>retrieve</i> and
210:      * <i>delete</i> an object from the current container by providing the object and/or
211:      * its local unique identifier ({@link kTAG_LID}).
212:      *
213:      * This method expects three parameters:
214:      *
215:      * <ul>
216:      *  <li><tt>$theObject</tt>: The object or the data to be modified.
217:      *  <li><tt>$theIdentifier</tt>: The local unique identifier ({@link kTAG_LID}) of the
218:      *      object. If the value is <i>NULL</i>, it means that it is the duty of the current
219:      *      container to set the value, this will generally be the case when inserting
220:      *      objects; in all other cases the parameter is required.
221:      *  <li><tt>$theModifiers</tt>: This parameter represents the operation options, it is a
222:      *      bitfield where the following values apply:
223:      *   <ul>
224:      *      <li>{@link kFLAG_PERSIST_INSERT}: The provided object will be inserted in the
225:      *          container. It is assumed that no other element in the container shares
226:      *          the same identifier, if not, the method should raise an exception.
227:      *          In this case the <tt>$theObject</tt> represents the full object and
228:      *          <tt>$theIdentifier</tt> represents the object unique identifier, or
229:      *          <tt>NULL</tt>, if it is the duty of the container to create the identifier.
230:      *          If the identifier was created by the container, it is the duty of this
231:      *          method to set it into the provided object.
232:      *          In this case the method should return the inserted object's native
233:      *          identifier ({@link kTAG_LID}).
234:      *      <li>{@link kFLAG_PERSIST_UPDATE}: The provided object will replace an
235:      *          object existing in the container. In this case the method expects the
236:      *          container to have an entry with the same key as the provided identifier,
237:      *          if this is not the case the method should raise an exception.
238:      *          In this case the <tt>$theObject</tt> represents the full object and
239:      *          <tt>$theIdentifier</tt> represents the object unique identifier.
240:      *          In this case the method should return <tt>TRUE</tt> if the object was
241:      *          updated and <tt>FALSE</tt> if the object was not found.
242:      *      <li>{@link kFLAG_PERSIST_REPLACE}: The provided object will be inserted
243:      *          if the identifier is missing or it doesn't match any entry in the container,
244:      *          or it will update the object corresponding to the provided identifier. This
245:      *          option represents the combination of the {@link kFLAG_PERSIST_INSERT} and
246:      *          {@link kFLAG_PERSIST_UPDATE} operations.
247:      *          In this case the <tt>$theObject</tt> represents the full object and
248:      *          <tt>$theIdentifier</tt> represents the object unique identifier, or
249:      *          <tt>NULL</tt>, if it is the duty of the container to create the identifier.
250:      *          If the identifier was created by the container, it is the duty of this
251:      *          method to set it into the provided object.
252:      *          In this case the method should return the inserted object's native
253:      *          identifier ({@link kTAG_LID}).
254:      *      <li>{@link kFLAG_PERSIST_MODIFY}: This option can be used to apply modifications
255:      *          to a subset of the object.
256:      *          In this case, <tt>$theObject</tt> should contain the modification
257:      *          <i>criteria</i> and <tt>$theIdentifier</tt> should contain the object's
258:      *          unique identifier, if the object cannot be located in the container, the
259:      *          method should not fail.
260:      *          In this case the method should set the modified object in the referenced
261:      *          parameter and return <tt>TRUE</tt> if the object was found and return
262:      *          <tt>FALSE</tt> if the object was not found.
263:      *      <li>{@link kFLAG_PERSIST_DELETE}: This option assumes you want to remove the
264:      *          object from the container.
265:      *          In this case the <tt>$theObject</tt> represents the full object and
266:      *          <tt>$theIdentifier</tt> represents the object unique identifier, what
267:      *          counts is that the object's unique identifier ({@link kTAG_LID}) is
268:      *          provided. If the object is not found in the container, the method should
269:      *          not fail.
270:      *          In this case the method should return <tt>TRUE</tt> if the object was
271:      *          deleted and <tt>FALSE</tt> if the object was not found.
272:      *   </ul>
273:      *      If none of the above flags are set, it means that the caller wants to retrieve
274:      *      the object identified by the {@link kTAG_LID} offset from the provided object
275:      *      object or from the provided identifier. If found, the provided object will
276:      *      receive the located object and the method will return <tt>TRUE</tt>; if not
277:      *      found, the method will set the provided object to <tt>NULL</tt> and return
278:      *      <tt>FALSE</tt>.
279:      * </ul>
280:      *
281:      * The method should raise an exception if the operation was not successful and update
282:      * the provided object with whatever data the operation will generate.
283:      *
284:      * The method will also raise an exception if the object is not yet initialised
285:      * ({@link _IsInited()}), this happens when the object received the native container.
286:      *
287:      * This class is abstract, which means that derived classes must implement the specific
288:      * functionality of specialised data stores, for this reason this method will only check
289:      * if the current object is inited.
290:      *
291:      * @param reference            &$theObject          Object.
292:      * @param mixed                 $theIdentifier      Identifier.
293:      * @param bitfield              $theModifiers       Options.
294:      *
295:      * @access public
296:      * @return mixed                The native operation status.
297:      *
298:      * @see kFLAG_PERSIST_INSERT kFLAG_PERSIST_UPDATE kFLAG_PERSIST_MODIFY
299:      * @see kFLAG_PERSIST_REPLACE kFLAG_PERSIST_MODIFY kFLAG_PERSIST_DELETE
300:      */
