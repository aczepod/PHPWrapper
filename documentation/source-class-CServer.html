 36: /**
 37:  * <h4>Server object ancestor</h4>
 38:  *
 39:  * This <i>abstract</i> class is the ancestor of all server classes in this library, it
 40:  * implements the interface and workflow that all concrete derived classes should implement.
 41:  *
 42:  * A server object holds a data member that represents a native connection to a server
 43:  * environment, server objects have the ability to generate other connection derived objects
 44:  * that have specific functionality, such as databases.
 45:  *
 46:  * The object is instantiated by providing the native server connection which is stored in a
 47:  * data member of the object, the array part of the object can be used to store specific
 48:  * attributes of the native server connection.
 49:  *
 50:  * The class also features a second property that can be used to store a concrete instance
 51:  * of the {@link CGraphContainer} class using the {@link Graph()} member accessor method.
 52:  * This property will be used by generated {@link CContainer} instances to write both to
 53:  * data containers and graphs.
 54:  *
 55:  * The abstract class must be overloaded by concrete classes that implement a native server
 56:  * object.
 57:  *
 58:  *  @package    MyWrapper
 59:  *  @subpackage Framework
 60:  */

148:      * The first parameter can take the following values:

151:      *  <li><tt>NULL</tt>: Return the current value.
152:      *  <li><tt>FALSE</tt>: Delete the current value.
