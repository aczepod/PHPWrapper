Class CDataWrapperClient

Wrapper client

This class represents a web-services wrapper client, it facilitates the job of requesting information from servers derived from the CDataWrapper class.

This class supports the following properties:

  • CDataWrapperClient::Database(): This element represents the web-service database name, it is stored in the in the kAPI_DATABASE offset.
  • CDataWrapperClient::Container(): This element represents the web-service container name, it is stored in the kAPI_CONTAINER offset.
  • CDataWrapperClient::PageStart(): This element represents the web-service page start, it is stored in the kAPI_PAGE_START offset.
  • CDataWrapperClient::PageLimit(): This element represents the web-service page limit, it is stored in the kAPI_PAGE_LIMIT offset.
  • CDataWrapperClient::Query(): This element represents the web-service query, it is stored in the kAPI_QUERY offset.
  • CDataWrapperClient::Select(): This element represents the web-service select fields, it is stored in the kAPI_SELECT offset.
  • CDataWrapperClient::Sort(): This element represents the web-service sort fields, it is stored in the kAPI_SORT offset.
  • CDataWrapperClient::Classname(): This element represents the web-service class name field, it represents the class name of the provided object and is stored in the kAPI_CLASS offset.
  • CDataWrapperClient::Object(): This element represents the web-service object, it is an array representing the object to be managed and is stored in the kAPI_OBJECT offset.

The class recognises the following operations (CDataWrapperClient::Operation()):

  • kAPI_OP_COUNT: This service returns the total number of elements affected by the provided query. The service expects the following parameters:
    • CWrapperClient::Format(): The data encoding type (kAPI_FORMAT).
    • CDataWrapperClient::Database(): The data encoding type (kAPI_DATABASE).
    • CDataWrapperClient::Container(): The data encoding type (kAPI_CONTAINER).
    • CDataWrapperClient::Query(): The query (kAPI_QUERY). If this parameter is omitted, it is assumed that the operation will involve all container elements.
  • kAPI_OP_GET: This service returns the elements that match the provided query. The service expects the following parameters:
    • CWrapperClient::Format(): The data encoding type (kAPI_FORMAT).
    • CDataWrapperClient::Database(): The data encoding type (kAPI_DATABASE).
    • CDataWrapperClient::Container(): The data encoding type (kAPI_CONTAINER).
    • CDataWrapperClient::Query(): The query (kAPI_QUERY). If this parameter is omitted, it is assumed that the operation will involve all container elements.
    • CDataWrapperClient::Select(): The list of requested attributes (kAPI_SELECT). If this parameter is omitted, it is assumed that all attributes are returned.
    • CDataWrapperClient::Sort(): The list of requested sort attributes (kAPI_SORT). If this parameter is omitted, it is assumed that the elements are returned in the order they were found.
    • CDataWrapperClient::PageLimit(): The maximum number of elements to be returned (kAPI_PAGE_LIMIT). If this parameter is not provided, the service will enforce the kDEFAULT_LIMIT value.
  • kAPI_OP_GET_ONE: This service returns the first element that matches the provided query. The service expects the following parameters:
    • CWrapperClient::Format(): The data encoding type (kAPI_FORMAT).
    • CDataWrapperClient::Database(): The data encoding type (kAPI_DATABASE).
    • CDataWrapperClient::Container(): The data encoding type (kAPI_CONTAINER).
    • CDataWrapperClient::Query(): The query (kAPI_QUERY). If this parameter is omitted, it is assumed that the operation will involve all container elements.
    • CDataWrapperClient::Select(): The list of requested attributes (kAPI_SELECT). If this parameter is omitted, it is assumed that all attributes are returned.
  • kAPI_OP_MATCH: This service returns the first element that matches the provided list of queries. The service expects the following parameters:
    • CWrapperClient::Format(): The data encoding type (kAPI_FORMAT).
    • CDataWrapperClient::Database(): The data encoding type (kAPI_DATABASE).
    • CDataWrapperClient::Container(): The data encoding type (kAPI_CONTAINER).
    • CDataWrapperClient::Query(): The query (kAPI_QUERY). If this parameter is omitted, it is assumed that the operation will involve all container elements.
    • CDataWrapperClient::Select(): The list of requested attributes (kAPI_SELECT). If this parameter is omitted, it is assumed that all attributes are returned.
  • kAPI_OP_INSERT: This service will insert the provided object into the provided database or container. The service expects the following parameters:
    • CWrapperClient::Format(): The data encoding type (kAPI_FORMAT).
    • CDataWrapperClient::Database(): The data encoding type (kAPI_DATABASE).
    • CDataWrapperClient::Container(): The data encoding type (kAPI_CONTAINER), this parameter may be omitted if the provided object class can resolve a default container from the provided database.
    • CDataWrapperClient::Object(): The object, (kAPI_OBJECT).
    • CDataWrapperClient::Classname(): The object class name, (kAPI_CLASS). This parameter is optional, if provided it will handle the persistence operation to the class.

Manage operation.

Manage operation.

We CWrapperClient::Operation() overload this method to add the following allowed operations:

  • kAPI_OP_COUNT: COUNT web-service operation, used to return the total number of elements satisfying a query.
  • kAPI_OP_MATCH: This operation is equivalent to a read query, except that it will try to match one kAPI_DATA_QUERY query clause at the time and will return a result on the first match.
  • kAPI_OP_GET kAPI_OP_GET: GET web-service operation, used to retrieve objects from the data store.
  • kAPI_OP_SET kAPI_OP_SET: SET web-service operation, used to insert or update objects in the data store.
  • kAPI_OP_UPDATE kAPI_OP_UPDATE: UPDATE web-service operation, used to update existing objects in the data store.
  • kAPI_OP_INSERT kAPI_OP_INSERT: INSERT web-service operation, used to insert new objects in the data store.

# Sort( string $theField = NULL, mixed $theOrder = NULL, boolean $getOld = FALSE )

Manage sort fields.

Manage sort fields.

This method can be used to manage the kAPI_SORT offset, the managed value is an array in which the key represents the attribute name and the value is an integer which determines the sort order: positive means ascending, negative means descending; zero is ignored.

  • $theField: The field name or list:
    • array: If you provide an array in this field, the method will assume you are providing a full list, in this case the method will set the offset with that value, without checking its contents.
    • NULL: Selects all fields.
    • other: Any other value will be interpreted as the field name.
  • $theOrder: The the sort order:
    • NULL: Return the sort order of the provided field. If the provided field is an array, this parameter will be ignored; if the provided field is a scalar, the method will return the element matching the provided field; if the provided field is NULL, the method will return the full list.
    • FALSE: Delete the element matching the provided field: if the provided field is an array, this parameter will be ignored; if the provided field is a scalar, the method will delete the element matching the provided field; if the provided field is NULL, the method will return the full list.
    • integer: Set the sort order, negative values mean descending; positive values mean ascending. If the provided field is an array, this parameter will be ignored; if the provided field is a scalar, the method will set the element matching the provided field; if the provided field is NULL, the method will set all the elements of the list to the provided sort order.
    • zero: If the parameter evaluates to zero, the operation will do nothing.
    • other: Any other type will be cast to an integer.
  • $getOld: This parameter is a boolean which if TRUE will return the old value when replacing values; if FALSE, it will return the currently set value.

Add a query statement.

Add a query statement.

This method can be used to add a query statement to the current query, it expects the query to be a scalar query with thew container provided in the service parameters.

The method expects the following parameters:

  • $theCondition: The kOPERATOR_AND, kOPERATOR_NAND, kOPERATOR_OR or kOPERATOR_NOR operator (see the Operators.inc.php source).
  • $theSubject: The subject or attribute of the query, it will be the tag kTAG_NID (see the Operators.inc.php source).
  • $theOperator: The statement operator or operation.
  • $theObject: The object or value of the statement, this parameter may be omitted of the operator does not require it.
  • $theType: The data type of the statement object (see the Types.inc.php source).

The query attribute will be set as a CQuery object and converted to an array before being sent to the server.

Note that this method should only be used if the kAPI_QUERY property of the object refers to a single query: if the attribute should refer to a list of queries, you should use the CDataWrapperClient::AddQueryListStatement() method.

# AddQueryListStatement( string $theIndex, string $theCondition, mixed $theSubject, string $theOperator, mixed $theObject = NULL, string $theType = NULL )

Add a query list statement.

Add a query list statement.

This method is similar to the CDataWrapperClient::AddQueryStatement() method as it adds a query statement to a query, except that the current CDataWrapperClient::Query() is a list of CQuery objects. This kind of structure is used by the kAPI_OP_MATCH operation to return the first match of a series of queries and by other operations in which the array element index represents the container name (CDataWrapperClient::Container()).

The method expects the following parameters:

  • $theIndex: This parameter represents the list index to the query to which the statement is to be appended. This parameter may either represent a generic index, or it may represent the container name to which the query applies.
  • $theCondition: The kOPERATOR_AND, kOPERATOR_NAND, kOPERATOR_OR or kOPERATOR_NOR operator (see the Operators.inc.php source).
  • $theSubject: The subject or attribute of the query, it will be the tag kTAG_NID (see the Operators.inc.php source).
  • $theOperator: The statement operator or operation.
  • $theObject: The object or value of the statement, this parameter may be omitted of the operator does not require it.
  • $theType: The data type of the statement object (see the Types.inc.php source).

Each element of the queries list will be a CQuery object and converted to an array before being sent to the server.

Note that this method should only be used if the kAPI_QUERY property of the object refers to a list of queries: if the attribute should refer to a scalar query, you should use the CDataWrapperClient::AddQueryStatement() method.
