Class CQueryStatement

Query statement

This class implements a query statement, such as those that populate the CQuery query class. This class concentrates on building and managing these statements, which are structured as follows:

  • kOFFSET_QUERY_SUBJECT: The query subject. It refers to the object element that we are considering in the query.
  • kOFFSET_QUERY_OPERATOR: The query predicate. This element represents the predicate or comparaison operator, it can take the following values:
    • kOPERATOR_DISABLED: Disabled, it means that the current statement is disabled.
    • kOPERATOR_EQUAL: Equality (=), this operator implies that the statement has also an kOFFSET_QUERY_DATA object.
    • kOPERATOR_EQUAL_NOT: Inequality (!=), negates the kOPERATOR_EQUAL operator; this operator implies that the statement has also a kOFFSET_QUERY_DATA object.
    • kOPERATOR_LIKE: Like, it is an accent and case insensitive equality filter, this operator implies that the statement has also a kOFFSET_QUERY_DATA object.
    • kOPERATOR_LIKE_NOT: The negation of the kOPERATOR_LIKE operator, this operator implies that the statement has also a kOFFSET_QUERY_DATA object.
    • kOPERATOR_PREFIX: Starts with, or prefix match, this operator implies that the statement has also a kOFFSET_QUERY_DATA object.
    • kOPERATOR_CONTAINS: Contains, selects all elements that contain the match value, this operator implies that the statement has also a kOFFSET_QUERY_DATA object.
    • kOPERATOR_SUFFIX: Ends with, or suffix match, this operator implies that the statement has also a kOFFSET_QUERY_DATA object.
    • kOPERATOR_REGEX: Regular expression, this operator implies that the statement has also a kOFFSET_QUERY_DATA object.
    • kOPERATOR_LESS: Smaller than (<), this operator implies that the statement has also a kOFFSET_QUERY_DATA object.
    • kOPERATOR_LESS_EQUAL: Smaller than or equal (<=), this operator implies that the statement has also a kOFFSET_QUERY_DATA object.
    • kOPERATOR_GREAT: Greater than (>), this operator implies that the statement has also a kOFFSET_QUERY_DATA object.
    • kOPERATOR_GREAT_EQUAL: Greater than or equal (>=), this operator implies that the statement has also a kOFFSET_QUERY_DATA object.
    • kOPERATOR_IRANGE: Range inclusive, matches >= value <=; this operator implies that the statement has also a kOFFSET_QUERY_DATA object which should contain the two range values.
    • kOPERATOR_ERANGE: Range exclusive, matches > value <, this operator implies that the statement has also a kOFFSET_QUERY_DATA object which should contain the two range values.
    • kOPERATOR_NULL: Is NULL or element is missing.
    • kOPERATOR_NOT_NULL:Not NULL or element exists.
    • kOPERATOR_IN: In, or belongs to set, this operator implies that the statement has also a kOFFSET_QUERY_DATA object that will contain the list of choices.
    • kOPERATOR_NI: Not in, the negation of kOPERATOR_IN, this operator implies that the statement has also a kOFFSET_QUERY_DATA object which contains the list of choices.
    • kOPERATOR_ALL: All, or match the full set, this operator implies that the statement has also a kOFFSET_QUERY_DATA object which will contain the set.
    • kOPERATOR_NALL: Not all, the negation of the kOPERATOR_ALL operator, this operator implies that the statement has also a kOFFSET_QUERY_DATA object which contains the set.
    • kOPERATOR_EX: Expression, indicates a complex expression, this operator implies that the statement has also a kOFFSET_QUERY_DATA object which will contain the expression.
  • kOFFSET_QUERY_TYPE : The statement object type, or data type of the kOFFSET_QUERY_DATA element, if the latter is required:
    • kTYPE_STRING: String, we assume in UTF8 character set.
    • kTYPE_INT32: 32 bit signed integer.
    • kTYPE_INT64: 64 bit signed integer.
    • kTYPE_FLOAT: Floating point number.
    • kTYPE_DATE: A date.
    • kTYPE_TIME: A date and time.
    • kTYPE_STAMP: A native timestamp.
    • kTYPE_BOOLEAN: An on/off switch.
    • kTYPE_BINARY: A binary string.
    • kTYPE_ENUM : An enumerated value.
    • kTYPE_ENUM_SET: An enumerated set of values.
  • kOFFSET_QUERY_DATA: The statement object or test data.

The main goal of this class is to provide an interface that may ease the construction of complex CQuery queries by providing specialised methods for building statements that can then safely be appended, CQuery::AppendStatement(), to CQuery objects.

Instantiate class

Instantiate class

The constructor can be used to instantiate a statement either by providing an existing statement structure, or by providing the statement elements:

  • $theSubject: The statement subject:
    • array or ArrayObject: Structures are interpreted as built statements, so the method will scan the structure and load the corresponding elements.
    • string: Any other type will be converted to a string and interpreted as the statement subject, or data element key.
  • $thePredicate: The statement operator or predicate:
    • kOPERATOR_DISABLED: Disabled, it means that the current statement is disabled; the remaining parameters are ignored.
    • kOPERATOR_EQUAL: Equality (=), this operator implies that the method expects the next two parameters.
    • kOPERATOR_EQUAL_NOT: Inequality (!=), negates the kOPERATOR_EQUAL operator; this operator implies that the method expects the next two parameters.
    • kOPERATOR_LIKE: Like, it is an accent and case insensitive equality filter, this operator implies that the method expects the next two parameters.
    • kOPERATOR_LIKE_NOT: The negation of the kOPERATOR_LIKE operator, this operator implies that the method expects the next two parameters.
    • kOPERATOR_PREFIX: Starts with, or prefix match, this operator implies that the method expects the next two parameters.
    • kOPERATOR_CONTAINS: Contains, selects all elements that contain the match string, this operator implies that the method expects the next two parameters.
    • kOPERATOR_SUFFIX: Ends with, or suffix match, this operator implies that the method expects the next two parameters.
    • kOPERATOR_REGEX: Regular expression, this operator implies that the method expects the next two parameters.
    • kOPERATOR_LESS: Smaller than (<), this operator implies that the method expects the next two parameters.
    • kOPERATOR_LESS_EQUAL: Smaller than or equal (<=), this operator implies that the method expects the next two parameters.
    • kOPERATOR_GREAT: Greater than (>), this operator implies that the method expects the next two parameters.
    • kOPERATOR_GREAT_EQUAL: Greater than or equal (>=), this operator implies that the method expects the next two parameters.
    • kOPERATOR_IRANGE: Range inclusive, matches >= value <=; this operator implies that the method expects the next three parameters.
    • kOPERATOR_ERANGE: Range exclusive, matches > value <, this operator implies that the method expects the next three parameters.
    • kOPERATOR_NULL: Is NULL or element is missing.
    • kOPERATOR_NOT_NULL:Not NULL or element exists.
    • kOPERATOR_IN: In, or belongs to set, this operator implies that the method expects the next two parameters.
    • kOPERATOR_NI: Not in, the negation of kOPERATOR_IN, this operator implies that the method expects the next two parameters.
    • kOPERATOR_ALL: All, or match the full set, this operator implies that the method expects the next two parameters.
    • kOPERATOR_NALL: Not all, the negation of the kOPERATOR_ALL operator, this operator implies that the method expects the next two parameters.
    • kOPERATOR_EX: Expression, indicates a complex expression, this operator implies that the method expects the next two parameters.
  • $theType: The statement object data type, this qualifies all remaining parameters. The allowed values are:
    • kTYPE_STRING: String, we assume in UTF8 character set, the string is expected in the next parameter.
    • kTYPE_INT32: 32 bit signed integer, the number is expected in the next parameter, either as an integer, float or string; once received, it will be converted to an int.
    • kTYPE_INT64: 64 bit signed integer, the number is expected in the next parameter, either as an integer, float or string; once received, it will be converted to an int.
    • kTYPE_FLOAT: Floating point number, the number is expected in the next parameter, either as an integer, float or string; once received, it will be converted to a float.
    • kTYPE_DATE: A string date, it is treated as a string date with a YYYYMMDD format in which month and day may be omitted.
    • kTYPE_TIME: A string time, it is treated as a string time with a YYYY-MM-DD HH:MM:SS format in which all elements are required; this element will be converted to a kTYPE_STAMP data type.
    • kTYPE_STAMP: A timestamp, optionally including microseconds.
    • kTYPE_BOOLEAN: An on/off switch, it will be converted to a 1/0 pair.
    • kTYPE_BINARY: A binary string.
  • $theObject: The statement object data, it should reflect the data type provided in the previous parameter.
  • $theRange: The statement range end or second element. This value must also reflect the data type provided in the previous parameter and will be automatically set if you provided a range and forgot to set it.

Manage subject

This method can be used to manage the query kOFFSET_QUERY_SUBJECT subject, it accepts a parameter which represents either the statement subject or the requested operation, depending on its value:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • other: Set the value with the provided parameter.

The second parameter is a boolean which if TRUE will return the old value when replacing values; if FALSE, it will return the currently set value.

Create a disabled statement

This method can be used to instantiate a disabled query statement. A disabled statement is one that should not execute, it can be used as a placeholder, or external methods may disable statements.

The statement uses the kOPERATOR_DISABLED operator and this method accepts the following parameters:

  • $theSubject: Statement subject.
  • $theObject: Statement object or first range bound.
  • $theType: Statement object data type.
  • $theRange: Statement second range bound.
