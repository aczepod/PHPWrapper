 36: /**
 37:  * <h3>Node object ancestor</h3>
 38:  *
 39:  * A node is an element of a tree or graph which is connected to other nodes through
 40:  * edges, a node is a vertex of a relationship. This class concentrates in the core
 41:  * features of the node, without taking into consideration the connections to other nodes.
 42:  *
 43:  * The node's main property is the term, {@link kOFFSET_TERM}, which defines the abstract
 44:  * concept that the node represents: a term taken by itself is an abstract concept, when it
 45:  * is referenced from a node which is related to other nodes through predicates, this term
 46:  * takes a different meaning, it becomes a term in context. Terms are like vocabulary, taken
 47:  * out of context they have a generalised meaning; put in context, they take a different
 48:  * meaning depending in what position of a term chain they are positioned. In this class,
 49:  * terms may hold any value.
 50:  *
 51:  * The node also features a {@link kOFFSET_KIND} property which is an enumerated set
 52:  * describing the specific type, or the specific function of the node in its chain or
 53:  * path.
 54:  *
 55:  * Finally, the class features the {@link kOFFSET_TYPE} property which defines the data type
 56:  * of the node or the data unit it represents.
 57:  *
 58:  * This class does not feature attributes that can represent the unique identifier of the
 59:  * object, this means that the container is responsible of providing the primary key of the
 60:  * object.
 61:  *
 62:  * Nodes can be considered {@link _IsInited()} when they have the  {@link kOFFSET_TERM}
 63:  * offset set. This and the other properties can be changed at any time, none are locked
 64:  * once the object has been committed: it is the responsibility of the caller or of derived
 65:  * classes to implement a locking rule.
 66:  *
 67:  * The class features member accessor methods for the default offsets:
 68:  *
 69:  * <ul>
 70:  *  <li>{@link Term()}: This method manages the node's term, {@link kOFFSET_TERM}.
 71:  *  <li>{@link Kind()}: This method manages the node's kind, {@link kOFFSET_KIND}.
 72:  *  <li>{@link Type()}: This method manages the node's type, {@link kOFFSET_TYPE}.
 73:  * </ul>
 74:  *
 75:  *  @package    MyWrapper
 76:  *  @subpackage Framework
 77:  */

174:     /**
175:      * <h4>Manage node kind set</h4>
176:      *
177:      * The node kind set, {@link kOFFSET_KIND}, holds a list of unique values that represent
178:      * the different kinds or types associated with the current node. The type,
179:      * {@link Type()}, of a node is a general qualification that applies to any class of
180:      * object, such as a data type; the kind, instead, refers to a qualification specific to
181:      * the current class of object.
182:      *
183:      * This offset collects the list of these qualifications in an enumerated set that can
184:      * be managed with the following parameters:
185:      *
186:      * <ul>
187:      *  <li><tt>$theValue</tt>: Depending on the next parameter, this may either refer to
188:      *      the value to be set or to the index of the element to be retrieved or deleted:
189:      *   <ul>
190:      *      <li><tt>NULL</tt>: This value indicates that we want to operate on all elements,
191:      *          which means, in practical terms, that we either want to retrieve or delete
192:      *          the full list. If the operation parameter resolves to <tt>TRUE</tt>, the
193:      *          method will default to retrieving the current list and no new element will
194:      *          be added.
195:      *      <li><tt>array</tt>: An array indicates that we want to operate on a list of
196:      *          values and that other parameters may also be provided as lists. Note that
197:      *          {@link ArrayObject} instances are not considered here as arrays.
198:      *      <li><i>other</i>: Any other type represents either the new value to be added or
199:      *          the index to the value to be returned or deleted.
200:      *   </ul>
201:      *  <li><tt>$theOperation</tt>: This parameter represents the operation to be performed
202:      *      whose scope depends on the value of the previous parameter:
203:      *   <ul>
204:      *      <li><tt>NULL</tt>: Return the element or full list.
205:      *      <li><tt>FALSE</tt>: Delete the element or full list.
206:      *      <li><tt>array</tt>: This type is only considered if the <tt>$theValue</tt>
207:      *          parameter is provided as an array: the method will be called for each
208:      *          element of the <tt>$theValue</tt> parameter matched with the corresponding
209:      *          element of this parameter, which also means that both both parameters must
210:      *          share the same count.
211:      *      <li><i>other</i>: Add the <tt>$theValue</tt> value to the list. If you provided
212:      *          <tt>NULL</tt> in the previous parameter, the operation will be reset to
213:      *          <tt>NULL</tt>.
214:      *   </ul>
215:      *  <li><tt>$getOld</tt>: Determines what the method will return:
216:      *   <ul>
217:      *      <li><tt>TRUE</tt>: Return the value <i>before</i> it was eventually modified.
218:      *      <li><tt>FALSE</tt>: Return the value <i>after</i> it was eventually modified.
219:      *   </ul>
220:      * </ul>
221:      *
222:      * @param mixed                 $theValue           Value or index.
223:      * @param mixed                 $theOperation       Operation.
224:      * @param boolean               $getOld             TRUE get old value.
225:      *
226:      * @access public
227:      * @return mixed                <i>New</i> or <i>old</i> native container.
228:      *
229:      * @uses ManageObjectSetOffset()
230:      *
231:      * @see kOFFSET_KIND
232:      */
233:     public function Kind( $theValue = NULL, $theOperation = NULL, $getOld = FALSE )
234:     {
235:         return ManageObjectSetOffset
236:             ( $this, kOFFSET_KIND, $theValue, $theOperation, $getOld );             // ==>
