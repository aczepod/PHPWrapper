Class CNode

Node object ancestor

A node is an element of a tree or graph which is connected to other nodes through edges, a node is a vertex of a relationship. This class concentrates in the core features of the node, without taking into consideration the connections to other nodes.

The class features the following default properties:

  • Term: The term, CNode::Term() or the kTAG_TERM tag, is a reference to a CTerm instance which holds the static attributes of the current node. This attribute is constituted by the kTAG_NID of the referenced term and is required.
  • Category: The category, CNode::Category() or the kTAG_CATEGORY tag, is an enumerated set of values that constitute the category or classification to which the object referred by the current node belongs to. This attribute is optional.
  • Kind: The kind, CNode::Kind() or the kTAG_KIND tag, is an enumerated set of values that define the function of the node. This attribute is optional.
  • Type: The type, CNode::Type() or the kTAG_TYPE tag, is an enumerated set of values that represent the type of the object defined by the current node. This attribute will be in general a combination of elements describing two main attribute categories: the data type and the cardinality. This attribute is optional.
  • Description: The description, CNode::Description() or the kTAG_DESCRIPTION tag, represents the description or long label of the node. This attribute is constituted by an array of elements in which the value of the element represents the description string and the element key is the language code in which the description is expressed in. This attribute is optional.

The node's main property is the term, kTAG_TERM, which defines the abstract concept that the node represents: a term taken by itself is an abstract concept, when it is referenced from a node which is related to other nodes through predicates, this term takes a different meaning, it becomes a term in context. Terms are like vocabulary, taken out of context they have a generalised meaning; put in context, they take a different meaning depending in what position of a term chain they are positioned. In this class, terms may hold any value.

The node also features a kTAG_KIND property which is an enumerated set describing the specific type, or the specific function of the node in its chain or path.

Finally, the class features the kTAG_TYPE property which defines the data type of the node or the data unit it represents, this offset is an enumerated set in which two elements can reside: the data type or unit, which is required, and optionally one or both kTYPE_REQUIRED and kTYPE_ARRAY which respectively indicate whether the node represents a required attribute and whether the data type indicated in the first element of the offset is a list or not.

This class does not feature attributes that can represent the unique identifier of the object, this means that the container is responsible of providing the primary key of the object.

Nodes can be considered CStatusDocument::_IsInited() when they have the kTAG_TERM offset set. This and the other properties can be changed at any time, none are locked once the object has been committed: it is the responsibility of the caller or of derived classes to implement a locking rule.

The class features member accessor methods for the default offsets:

  • CNode::Term(): This method manages the node's term, kTAG_TERM.
  • CNode::Kind(): This method manages the node's kind, kTAG_KIND.
  • CNode::Type(): This method manages the node's type, kTAG_TYPE.

# PID( mixed $theValue = NULL, boolean $getOld = FALSE )

Manage persistent identifier

Manage persistent identifier

The persistent identifier, kTAG_PID, holds a string value which represents the object's persistent identifier. This value should uniquely identify the object across implementations and should not change once the object has been committed.

This attribute becomes necessary when the object does not have a global unique identifier and its native identifier is not persistent. In nodes this is true because the CNode::Term(), which is its global identifier, is not unique and its native identifier, CPersistentDocument::NID(), is a non persistent sequence number: this attribute can be used to set a value which can be used to differentiate nodes which point to the same term and that may have different native identifiers in different implementations.

The method accepts a parameter which represents either the new value or the requested operation, depending on its type:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • other: Set the value with the provided parameter.

The second parameter is a boolean which if TRUE will return the old value when replacing an existing value; if FALSE, it will return the currently set value.

Note that, while this method allows the creation, modification and deletion of this property, the hosting object may prevent some of these actions, by default this attribute is locked when the object has the CStatusDocument::_IsCommitted() status.

# Description( mixed $theLanguage = NULL, mixed $theValue = NULL, boolean $getOld = FALSE )

Manage node description

Manage node description

The node description, kTAG_DESCRIPTION, represents the node's long label or extended definition. It is an optional attribute of the object that holds an array of elements in which the index is represented by the language code and the value is the string.

No two elements may share the same language code.

The description depends on the context in which the object is, as opposed as the definition which does not depend on the context.

The method accepts the following parameters:

  • $theLanguage: Language code.
  • $theValue: The description string or the operation, depending on its value:
    • NULL: Return the string corresponding to the provided language.
    • FALSE: Delete the element corresponding to the provided language.
    • other: Any other value represents the description string that will be set or replace the entry for the provided language.
  • $getOld: If TRUE, the method will return the description string before it was eventually modified; if FALSE, the method will return the value after eventual modifications.

# RelateTo( mixed $thePredicate, mixed $theObject, CConnection $theConnection = NULL, boolean $doPropagate = TRUE )

Create an outgoing edge

Create an outgoing edge

This method can be used to instantiate an outgoing edge by providing the object vertex node and edge predicate. The subject vertex is represented by the current node.

Both the predicate and the object vertex node can be any type, the last parameter represents the connection, it is optional and if provided, will be used to commit the edge.

The method expects the following parameters:

  • $theObject: The edge's object vertex.
  • $thePredicate: The edge's predicate.
  • $theConnection: Server, database or container.
  • $doPropagate: This boolean switch indicates whether the relationship is to be propagated. This feature is not used in this class, but derived classes will use it; the default value is TRUE.

The method will return an instance of the CEdge class, if any error occurs, the method will raise an exception.
