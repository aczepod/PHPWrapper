 57: /**
 58:  * <h3>Persistent objects data store ancestor</h3>
 59:  *
 60:  * This <i>abstract</i> class is the ancestor of all container classes in this library, it
 61:  * implements the interface and workflow that all concrete derived classes should implement.
 62:  *
 63:  * A container object serves the purpose of storing and retrieving objects, in the same way
 64:  * as a database table. The object holds a data member that represents a native container
 65:  * from a database.
 66:  *
 67:  * The public interface declares the following main operations:
 68:  *
 69:  * <ul>
 70:  *  <li>{@link ManageObject()}: This method will <i>insert</i>, <i>update</i>,
 71:  *      <i>modify</i>, <i>delete</i> and <i>load</i> an object from the current container.
 72:  *      This method operates one object at the time and it is mainly used by persistent
 73:  *      objects for persisting in containers.
 74:  *  <li>{@link ConvertValue()}: This method will convert a PHP value to a container native
 75:  *      value.
 76:  * </ul>
 77:  *
 78:  * The class does not implement a concrete data store, derived classes must implement
 79:  * specific storage actions.
 80:  *
 81:  *  @package    MyWrapper
 82:  *  @subpackage Framework
 83:  */

 97:      *  ManageObject                                                                    *
 98:      *==================================================================================*/
 99: 
100:     /**
101:      * <h4>Manage an object from the container</h4>
102:      *
103:      * This method can be used to manage a single object in the container, it can
104:      * <i>insert</i>, <i>update</i>, <i>replace</i>, <i>modify</i>, <i>retrieve</i> and
105:      * <i>delete</i> an object from the current container by providing the object and/or
106:      * its native unique identifier ({@link kOFFSET_NID}).
107:      *
108:      * This method expects three parameters:
109:      *
110:      * <ul>
111:      *  <li><tt>$theObject</tt>: The object or the data to be modified.
112:      *  <li><tt>$theIdentifier</tt>: The native unique identifier ({@link kOFFSET_NID}) of
113:      *      the object. If the value is <i>NULL</i>, it means that it is the duty of the
114:      *      current container to set the value, this will generally be the case when
115:      *      inserting objects; in all other cases the parameter is required.
116:      *  <li><tt>$theModifiers</tt>: This parameter represents the operation options, or the
117:      *      list of offsets to be returned. If the parameter is not an array, it will be
118:      *      interpreted as a bitfield where the following values apply:
119:      *   <ul>
120:      *      <li>{@link kFLAG_PERSIST_INSERT}: The provided object will be inserted in the
121:      *          container. It is assumed that no other element in the container shares
122:      *          the same identifier, if not, the method should raise an exception.
123:      *          In this case the <tt>$theObject</tt> represents the full object and
124:      *          <tt>$theIdentifier</tt> represents the object unique identifier, or
125:      *          <tt>NULL</tt>, if it is the duty of the container to create the identifier.
126:      *          If the identifier was created by the container, it is the duty of this
127:      *          method to set it into the provided object.
128:      *          In this case the method should return the inserted object's native
129:      *          identifier ({@link kOFFSET_NID}).
130:      *      <li>{@link kFLAG_PERSIST_UPDATE}: The provided object will replace an
131:      *          object existing in the container.
132:      *          In this case the <tt>$theObject</tt> represents the full object and
133:      *          <tt>$theIdentifier</tt> represents the object unique identifier.
134:      *          In this case the method should return <tt>TRUE</tt> if the object was
135:      *          updated and <tt>FALSE</tt> if the object was not found.
136:      *      <li>{@link kFLAG_PERSIST_REPLACE}: The provided object will be inserted
137:      *          if the identifier is missing or it doesn't match any entry in the container,
138:      *          or it will update the object corresponding to the provided identifier. This
139:      *          option represents the combination of the {@link kFLAG_PERSIST_INSERT} and
140:      *          {@link kFLAG_PERSIST_UPDATE} operations.
141:      *          In this case the <tt>$theObject</tt> represents the full object and
142:      *          <tt>$theIdentifier</tt> represents the object unique identifier, or
143:      *          <tt>NULL</tt>, if it is the duty of the container to create the identifier.
144:      *          If the identifier was created by the container, it is the duty of this
145:      *          method to set it into the provided object.
146:      *          In this case the method should return the inserted object's native
147:      *          identifier ({@link kOFFSET_NID}).
148:      *      <li>{@link kFLAG_PERSIST_MODIFY}: This option can be used to apply modifications
149:      *          to a subset of the object.
150:      *          In this case the parameters of this method have the following functions:
151:      *       <ul>
152:      *          <li><tt>$theObject</tt>: This parameter will be an array of elements whose
153:      *              <i>key</i> corresponds to the offset to be considered and the
154:      *              <i>value</i> to the value that the operation will use.
155:      *          <li><tt>$theIdentifier</tt>: This parameter keeps its original meaning, it
156:      *              should contain the native identifier of the object to be modified.
157:      *          <li><tt>$theModifiers</tt>: Another section of this bitfield will hold the
158:      *              flags that determine what specific modification should be performed:
159:      *           <ul>
160:      *              <li>{@link kFLAG_MODIFY_INCREMENT}: This flag indicates an increment or
161:      *                  decrement operation, the method will take the value corresponding
162:      *                  to the offset in <tt>$theObject</tt> parameter and add it to the
163:      *                  object identified by <tt>$theIdentifier</tt> in the container.
164:      *                  For instance, to increment by one the '<tt>A</tt>' offset of object
165:      *                  <tt>X</tt>, you would pass the <tt>array( 'A' => 1 )</tt> in
166:      *                  <tt>$theObject</tt>, <tt>X</tt> in <tt>$theIdentifier</tt> and set
167:      *                  the kFLAG_MODIFY_INCREMENT flag; to decrement by two you would
168:      *                  pass <tt>array( 'A' => -2 )</tt> in <tt>$theObject</tt>, with all
169:      *                  other parameters unchanged.
170:      *                  If the offset does not exist, it will be initialised with the
171:      *                  increment value.
172:      *              <li>{@link kFLAG_MODIFY_APPEND}: This flag indicates that we want to
173:      *                  append a value to an existing array, the method will take the value
174:      *                  corresponding to the offset in <tt>$theObject</tt> and append it to
175:      *                  the offset in <tt>$theObject</tt> for the object identified by
176:      *                  <tt>$theIdentifier</tt> in the container.
177:      *                  For instance, to append the value '<tt>A</tt>' to the array in
178:      *                  offset '<tt>FIELD</tt>' for object <tt>X</tt>, you would pass the
179:      *                  <tt>array( 'FIELD' => 'A' )</tt> in <tt>$theObject</tt>,
180:      *                  <tt>X</tt> in <tt>$theIdentifier</tt> and set the
181:      *                  kFLAG_MODIFY_APPEND flag.
182:      *                  If the field does not exist, it will be created with an array
183:      *                  composed of the provided append value; if the field exists and its
184:      *                  value is not an array, an exception should be raised.
185:      *              <li>{@link kFLAG_MODIFY_ADDSET}: This flag indicates that we want to
186:      *                  add a value to a set, this operation is equivalent to
187:      *                  {@link kFLAG_MODIFY_APPEND}, except that the value will be appended
188:      *                  only if it doesn't exist already in the receiving array.
189:      *              <li>{@link kFLAG_MODIFY_POP}: This flag indicates that we want to remove
190:      *                  the first or last element from an array. The method will take the
191:      *                  value corresponding to the offset in <tt>$theObject</tt> and check
192:      *                  its sign: if positive, the method will remove the first element; if
193:      *                  negative it will remove the last.
194:      *                  For instance, to remove the first element in the '<tt>FIELD</tt>'
195:      *                  offset of object <tt>X</tt>, you would pass the
196:      *                  <tt>array( 'FIELD' => 1 )</tt> in <tt>$theObject</tt>, <tt>X</tt> in
197:      *                  <tt>$theIdentifier</tt> and set the kFLAG_MODIFY_POP flag in
198:      *                  <tt>$theModifiers</tt>; to remove the last element you would pass
199:      *                  <tt>array( 'FIELD' => -1 )</tt> in <tt>$theObject</tt>.
200:      *                  If the offset does not exist, the method should not fail; if the
201:      *                  offset is not an array, the method should raise an exception.
202:      *              <li>{@link kFLAG_MODIFY_PULL}: This flag indicates that we want to
203:      *                  remove all occurrences of a value from an array.
204:      *                  For instance, to remove all occurrances of '<tt>A</tt>' from the
205:      *                  array contained in the '<tt>FIELD</tt>' offset of object <tt>X</tt>, 
206:      *                  you would pass <tt>array( 'FIELD' => 'A' )</tt> in
207:      *                  <tt>$theObject</tt>, <tt>X</tt> in <tt>$theIdentifier</tt> and set
208:      *                  the kFLAG_MODIFY_PULL flag in <tt>$theModifiers</tt>.
209:      *                  If the field exists and its value is not an array, an exception
210:      *                  should be raised.
211:      *              <li>{@link kFLAG_MODIFY_MASK} <i>off</i>: If none of the above flags are
212:      *                  set, it means that the key/value pairs set in <tt>$theObject</tt>
213:      *                  represent offsets to be added or removed, depending on the value
214:      *                  part of the pair: if the value is <tt>NULL</tt>, it means that you
215:      *                  want to remove the offset; any other value will replace the existing
216:      *                  offset or be added to the object.
217:      *           </ul>
218:      *       </ul>
219:      *          In this case the method will return the modified object in the
220:      *          <tt>$theObject</tt> parameter, or raise an exception if the operation fails.
221:      *      <li>{@link kFLAG_PERSIST_DELETE}: This option assumes you want to remove the
222:      *          object from the container.
223:      *          In this case the <tt>$theObject</tt> represents the full object and
224:      *          <tt>$theIdentifier</tt> represents the object unique identifier, what
225:      *          counts is that the object's unique identifier ({@link kOFFSET_NID}) is
226:      *          provided. If the object is not found in the container, the method should
227:      *          not fail.
228:      *          In this case the method should return <tt>TRUE</tt> if the object was
229:      *          deleted and <tt>FALSE</tt> if the object was not found.
230:      *   </ul>
231:      *      If none of the above flags are set, it means that the caller wants to retrieve
232:      *      the object identified by the {@link kOFFSET_NID} offset from the provided object
233:      *      or from the provided identifier. If found, the provided object will receive the
234:      *      located object and the method will return <tt>TRUE</tt>; if not found, the
235:      *      method will set the provided object to <tt>NULL</tt> and return <tt>FALSE</tt>.
236:      *      If the parameter is an array, it implies that the requested operation is to
237:      *      retrieve the object, the array will be interpreted as the list of object offsets
238:      *      to be returned, the {@link kOFFSET_NID} offset is returned by default.
239:      * </ul>
240:      *
241:      * The method should raise an exception if the operation was not successful and update
242:      * the provided object with whatever data the operation will generate.
243:      *
244:      * The method will also raise an exception if the object is not yet initialised
245:      * ({@link _IsInited()}), this happens when the object received the native container.
246:      *
247:      * This class is abstract, which means that derived classes must implement the specific
248:      * functionality of specialised data stores, for this reason this method is declared
249:      * abstract.
250:      *
251:      * @param reference            &$theObject          Object.
252:      * @param mixed                 $theIdentifier      Identifier.
253:      * @param mixed                 $theModifiers       Options or offsets list.
254:      *
255:      * @access public
256:      * @return mixed                The native operation status.
257:      *
258:      * @see kFLAG_PERSIST_INSERT kFLAG_PERSIST_UPDATE kFLAG_PERSIST_MODIFY
259:      * @see kFLAG_PERSIST_REPLACE kFLAG_PERSIST_MODIFY kFLAG_PERSIST_DELETE
260:      * @see kFLAG_MODIFY_INCREMENT kFLAG_MODIFY_APPEND kFLAG_MODIFY_ADDSET
261:      * @see kFLAG_MODIFY_POP kFLAG_MODIFY_PULL kFLAG_MODIFY_MASK
262:      */
263:     abstract public function ManageObject( &$theObject,
264:                                             $theIdentifier = NULL,
265:                                             $theModifiers = kFLAG_DEFAULT );

323:     /**
324:      * <h4>Return a sequence number</h4>
325:      *
326:      * This method should return a sequence number connected to the provided key. Each time
327:      * this method is called, the sequence number is incremented, which means that you
328:      * should only call it when you intend to use this number.
329:      *
330:      * The first parameter, <tt>$theKey</tt>, represents the key to the sequence, it will
331:      * be the {@link kOFFSET_NID} of the record holding the sequence and the field holding
332:      * the current number has the {@link kOFFSET_SEQUENCE} offset.
333:      *
334:      * The second parameter represents the container in which the sequence is stored:
335:      *
336:      * <ul>
337:      *  <li><tt>string<tt>: The method should use the native container corresponding to the
338:      *      provided string.
339:      *  <li><tt>TRUE<tt>: The method should use the string stored in
340:      *      {@link kCONTAINER_SEQUENCE_NAME}.
341:      *  <li><tt>NULL<tt>: The method should use the current container.
342:      * </ul>
343:      *
344:      * If the object is not {@link _Is Inited()}, the method should raise an exception.
345:      *
346:      * If the current object does not support sequences, this method should return
347:      * <tt>NULL</tt>.
348:      *
349:      * This class is abstract and by default sequences are not supported.
350:      *
351:      * @param string                $theKey             Sequence key.
352:      * @param mixed                 $theContainer       Sequence container.
353:      *
354:      * @access public
355:      * @return mixed                The sequence number or <tt>NULL</tt>.
356:      *
357:      * @throws Exception
358:      */
