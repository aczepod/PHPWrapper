Class CUser

User object ancestor

A default user contains the following attributes:

  • User name: The user full name, CUser::Name() or the kTAG_USER_NAME tag, is a string that represents the full user name. In this class we make no distinction between first and last names.
  • User code: The user code, CUser::Code() or the kTAG_USER_CODE tag, is a string that represents the user code or identifier. This string must be unique among all users, it represents the user's global identifier and it will be used as the user's native identifier.
  • User password: The user password, CUser::Pass() or the kTAG_USER_PASS tag, is a string that represents the user password: along with the user code, it represents the login credentials.
  • User mail: The user e-mail, CUser::Mail() or the kTAG_USER_MAIL tag, is a string that represents the user e-mail address, it could generally be used to initialise the user code.
  • User role: The user role, CUser::Role() or the kTAG_USER_ROLE tag, is a string that represents the user role or credentials, it is a system dependent element that provides the host system with the necessary information to provide the user the correct privileges.
  • User profile: The user profile, CUser::Profile() or the kTAG_USER_PROFILE tag, is a set of elements that list the capabilities or credentials of the user. The role is specific to the hosting system, while the profile is specific to this library.
  • User manager: The user manager, CUser::Manager() or the kTAG_USER_MANAGER tag, is a reference to the user that has the rights and responsibility of managing the user. It will be generally the one who created the user, who can revoke privileges or delete the user.

The native identifier of the user, kTAG_NID, is equivalent to the user code, in derived classes the native identifier may change.

Once the object has been committed, the user code will be locked, since it is used to generate the unique identifier.

The object will have its CStatusDocument::_IsInited() status set if the name, code, password and mail are also set.

Load manager object

Load manager object

This method will return the current manager object: if the term is not set, the method will return NULL; if the manager cannot be found, the method will raise an exception.

The object will also be loaded in a data member that can function as a cache.

The method features two parameters: the first refers to the container in which the manager is stored, the second is a boolean flag that determines whether the object is to be read, or if the cached copy can be used.

# Resolve( CConnection $theConnection, mixed $theIdentifier, boolean $doThrow = FALSE )

Resolve a user

Resolve a user

This method can be used to locate a user given the attributes that comprise its identifier.

The method accepts the following parameters:

  • $theConnection: This parameter represents the connection from which the terms container must be resolved. If this parameter cannot be correctly determined, the method will raise an exception.
  • $theIdentifier: In this class by default the native identifier is the user code, we assume this parameter is that string.
  • $doThrow: If TRUE, any failure to resolve the term or its namespace, will raise an exception.

The method will return the found user, NULL if not found, or raise an exception if the last parameter is TRUE.

Note: do not provide an array containing the object in the identifier parameter, or you will get unexpected results.
