Class CWrapperClient

Wrapper client

This class represents a web-services wrapper client, it facilitates the job of requesting information from servers derived from the CWrapper class.

This class supports the following properties:

  • CWrapperClient::Operation(): This element represents the web-service requested operation, it is stored in the kAPI_OPERATION offset.
  • CWrapperClient::Format(): This element represents the web-service parameters and response format, it is stored in the kAPI_FORMAT offset.
  • CWrapperClient::Stamp(): This element represents a switch to turn on and off timing information from the web service, it is stored in the kAPI_STAMP_REQUEST offset.
  • Log CWrapperClient::LogRequest(): This element represents a switch to turn onand off logging the request: if on, the request will be returned by the web-service, it is stored in the kAPI_LOG_REQUEST offset.
  • Log CWrapperClient::LogTrace(): This element represents a switch to turn on and off tracing exceptions: if on, any exception will also hold a trace, it is stored in the kAPI_LOG_TRACE offset.

The class recognises the following operations (CWrapperClient::Operation()):

  • kAPI_OP_HELP: This service returns the list of supported operations and options. The service expects the following parameters:
    • CWrapperClient::Format(): The data encoding type (kAPI_FORMAT).
  • kAPI_OP_PING: This service returns a status response. The service expects the following parameters:
    • CWrapperClient::Format(): The data encoding type (kAPI_FORMAT).

Objects of this class require at least the CConnection::Connection(), CWrapperClient::Operation() and CWrapperClient::Format() to be set if they expect to have an CStatusDocument::_IsInited() status, which is necessary to send requests to web-services.

# Operation( string $theValue = NULL, boolean $getOld = FALSE )

Manage operation.

Manage operation.

This method can be used to manage the kAPI_OPERATION operation, it accepts a parameter which represents either the web-service operation code or this method operation, depending on its value:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • other: Set the value with the provided parameter.

The second parameter is a boolean which if TRUE will return the old value when replacing values; if FALSE, it will return the currently set value.

This method will check whether the provided operation is supported, if this is not the case, it will raise an exception. Derived classes should first check their specific operations, if not matched, they should pass the parameter to the parent method. In this class we support the following operations:

  • kAPI_OP_HELP kAPI_OP_HELP: HELP web-service operation, which returns the list of supported operations and options.
  • kAPI_OP_PING kAPI_OP_PING: PING web-service operation, which returns a status response.

Manage format.

Manage format.

This method can be used to manage the kAPI_FORMAT format in which both the parameters are sent and the response is returned from the web-service. It accepts a parameter which represents either the web-service format code or this method operation, depending on its value:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • other: Set the value with the provided parameter.

The second parameter is a boolean which if TRUE will return the old value when replacing values; if FALSE, it will return the currently set value.

This method will check whether the provided format is supported, if this is not the case, it will raise an exception. Derived classes should first check their specific operations, if not matched, they should pass the parameter to the parent method. In this class we support the following operations:

  • kTYPE_PHP: Parameters and response will be serialized.
  • kTYPE_XML: Parameters and response will be encoded in XML.
  • kTYPE_JSON: Parameters and response will be encoded in JSON.
  • kTYPE_META: This data type can be used to return service metadata, the Execute() request will return headers metadata for troubleshooting, rather than the response from the web-service.

Send a HTTP request.

Send a HTTP request.

This method can be used to sent an HTTP request via GET or POST.

The method accepts the following parameters:

  • $theUrl: The request URL.
  • $theParams: An array of key/value parameters for the request.

    • kTYPE_JSON: JSON.
    • kTYPE_META: Metadata: the method will return the metadata of the operation for troubleshooting purposes.

Encode parameters.
