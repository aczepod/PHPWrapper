Class COntologyEdge

Ontology edge object ancestor

This class extends its ancestor, CEdge, by asserting the CEdge::Subject() and CEdge::Object() to be instances of COntologyNode, and the CEdge::Predicate() to be an instance of COntologyTerm.

This class overloads the native identification workflow by using a number sequence tagged by kSEQUENCE_KEY_EDGE. The class also features a unique identifier, kOFFSET_UID, which receives the hash of the global identifier, this will be useful to identify duplicates.

When adding vertices and predicates to the object, these can be provided as objects and if these objects are not CStatusDocument::_IsCommitted(), they will be stored before the current edge object is committed.

The class features an offset, kOFFSET_REFS_TAG, which represents the list of tags that reference the current node. This offset is a set of tag identifiers implemented as an array. The offset definition is borrowed from the COntologyTerm class, which is required by this class because of its kOFFSET_PREDICATE offset. This offset is managed by the tag class, this class locks the offset.

The class implements the static method, COntologyEdge::DefaultContainer(), it will use the kCONTAINER_EDGE_NAME constant. Note that when passing CConnection based objects to the persisting methods of this class, you should provide preferably Database instances, since this class may have to commit terms.

The class features member accessor methods for the default offsets:

  • COntologyEdge::TagRefs(): This method returns the node's tag references, kOFFSET_REFS_TAG.
  • COntologyEdge::LoadSubject(): This method will return the referenced subject vertex COntologyNode object.
  • COntologyEdge::LoadPredicate(): This method will return the referenced predicate COntologyTerm object.
  • COntologyEdge::LoadObject(): This method will return the referenced object vertex COntologyNode object.

# _PrecommitIdentify( reference & $theConnection, reference & $theModifiers )

Determine the identifiers before committing

Determine the identifiers before committing

This method will use the result of the COntologyEdge::_index() method to set the global identifier, kOFFSET_GID; the same value will be hashed and constitute the unique identifier, kOFFSET_UID.

Objects of this class are identified by a sequence number tagged kSEQUENCE_KEY_EDGE, this method will set the native identifier, kOFFSET_NID, with this value.

The parent method will then be called, which will ignore the global and native identifiers, since they will have been set here.

# _ReferenceInSubject( CConnection $theConnection, boolean $doAdd )

Add edge reference to subject vertex

Add edge reference to subject vertex

This method can be used to add or remove the current edge's reference, kOFFSET_REFS_EDGE, from the subject vertex node, kOFFSET_VERTEX_SUBJECT. This method should be used whenever committing a new edge or when deleting one: it will add the current edge's native identifier to the set of edge references of the edge's subject vertex when committing a new edge; it will remove it when deleting the edge.

The last parameter is a boolean: if TRUE the method will add to the set; if FALSE, it will remove from the set.

The method will return TRUE if the operation affected at least one object, FALSE if not, NULL if the subject vertex is not set and raise an exception if the operation failed.
