Class COntologyWrapperClient

Ontology wrapper client

This class represents a web-services ontology wrapper client, it facilitates the job of requesting information from servers derived from the COntologyWrapper class.

This class supports the following properties:

  • COntologyWrapperClient::Language(): This element represents the web-service language code, it is stored in the in the kAPI_LANGUAGE offset.
  • COntologyWrapperClient::Predicate(): This element represents the web-service predicate reference, it is stored in the kAPI_PREDICATE offset.
  • COntologyWrapperClient::Relations(): This element represents the web-service relations sense code, it is stored in the kAPI_RELATION offset.

The class recognises the following operations (COntologyWrapperClient::Operation()):

  • kAPI_OP_GetVertex: This service will first select the list of vertices matching the provided query, if the kAPI_RELATION parameter is provided, the service will use the first found vertex and return the list of vertices related to that vertex, depending on the value of the kAPI_RELATION parameter:
    • kAPI_RELATION_IN: Return all vertices that point to the vertex.
    • kAPI_RELATION_OUT: Return all vertices to which the vertex points to.
    • kAPI_RELATION_ALL: Return all vertices related to the vertex.
    If the kAPI_RELATION parameter is missing, the service will return the matched list of vertices.

# Language( string $theValue = NULL, boolean $getOld = FALSE )

Manage language code(s).

Manage language code(s).

This method can be used to manage the kAPI_LANGUAGE offset, it accepts a parameter which represents the list of language codes in which kTYPE_LSTRING attributes are set, or the requested operation, depending on its value:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • other: Set the value with the provided parameter, in this case the value is expected to be:
    • array or ArrayObject: The provided parameter will become the new list.
    • other: The method will assume the parameter is a string and it will add it to the current list or create a list if it doesn't exist.

The second parameter is a boolean which if TRUE will return the old value when replacing values; if FALSE, it will return the currently set value.

# Relations( string $theValue = NULL, boolean $getOld = FALSE )

Manage relationships sense.

Manage relationships sense.

This method can be used to manage the kAPI_RELATION offset, it accepts a parameter which represents the relationships sense, or the requested operation, depending on its value:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • kAPI_RELATION_IN: Incoming relationships.
  • kAPI_RELATION_OUT: Outgoing relationships.
  • kAPI_RELATION_ALL: Incoming and outgoing relationships.
  • other: The method will raise an exception.

The second parameter is a boolean which if TRUE will return the old value when replacing values; if FALSE, it will return the currently set value.
