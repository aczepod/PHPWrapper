CMongoContainer::__construct (   $theConnection = NULL,
  $theOptions = NULL 
)

Instantiate class

The method accepts the same parameters as the MongoCollection constructor, the first one represents the database object, the second represents the container name, if the latter is missing, the method will raise an exception.

The method will also accept a database object in the first parameter, in that case the second parameter will be ignored.

The method will first attempt to instantiate the MongoCollection object, it will then store it in the connection data member.

CMongoContainer::UnserialiseData ( &  $theElement,
  $theType = NULL 
)

Unserialise provided data element.

This method should convert the provided structure into a custom data type compatible with the current container.

This method is called by a public UnserialiseObject() interface which traverses an object and provides this method with all elements that satisfy the following conditions:

  • CDataType: All instances derived from this class are sent to this method.
  • Array or ArrayObject: If the structure is composed of exactly two offsets and these elements are kTAG_CUSTOM_TYPE and kTAG_CUSTOM_DATA, it will be sent to this method.

In this class we parse the following types and offsets:

  • CDataTypeMongoId object or kTYPE_MongoId offset: We return a MongoId object.
  • CDataTypeMongoCode object or kTYPE_MongoCode offset: We return a MongoCode object.
  • CDataTypeStamp object or kTYPE_STAMP offset: We return a MongoDate object.
  • CDataTypeRegex object or kTYPE_REGEX_STRING offset: We return a MongoRegex object.
  • CDataTypeInt32 object or kTYPE_INT32 offset: We return a MongoInt32 object.
  • CDataTypeInt64 object or kTYPE_INT64 offset: We return a MongoInt64 object.
  • CDataTypeBinary object or kTYPE_BINARY_STRING offset: We return a MongoBinData object.

The elements to be converted are provided by reference, which means that they have to be converted in place.

This method can also be used in a different way: you can ask the method to convert the provided scalar to the corresponding Mongo custom type, for this you need to provide a scalar in the first parameter and a data type in the second.
