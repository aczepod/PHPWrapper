COntologyMasterNode::_PrecommitIdentify ( &  $theConnection,
&  $theModifiers 
)
protected

Determine the identifiers before committing

We overload this method to prevent creating two nodes that refer to the same term. We first check if there is another node that refers to the same term and that does point to another node: if we find one we add the kTAG_CATEGORY, kTAG_KIND, kTAG_TYPE, kTAG_INPUT and kTAG_EXAMPLES attributes to the found node and make it the current node; if no such node is found, we call the parent method.

In the first case we return the found node identifier, this will stop the commit workflow.

  $theConnection = NULL,

)

Relate to node

We overload this method to mirror relationships between alias nodes to their counterpart master nodes, this is governed by the $doPropagate switch.

We first create the requested relationship, then, if the object is an alias, we relate the current object with the object's master.

This means that:

  • MASTER => MASTER:
    • MASTER => MASTER
  • MASTER => ALIAS:
    • MASTER => ALIAS
    • MASTER => ALIAS.MASTER

Note that this method is called by its counterpart in COntologyAliasNode, this means

We require the connection parameter here since both subject and object vertices must be committed by this method, this implies that the edge will also be committed.

static COntologyMasterNode::Resolve ( CConnection  $theConnection,
  $theIdentifier,

)
static

Resolve a node

We overload this method in this class by intercepting all cases in which the $theIdentifier is not an integer, or node reference: since no two nodes of this class can share the same term reference, we add a clause to the search query to exclude all node that have a kTAG_NODE attribute. We do this rather than use the object class, because we assume all nodes that have the kTAG_NODE attribute to be aliases.

Since terms become also a unique identifier, this method will return either an object or NULL.
