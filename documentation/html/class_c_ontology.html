Create template

This method will check whether the received node corresponds to a tag feature, in that case it will add the tag's info to the provided template array, if not, it will recurse on all nodes related to the provided node by the provided predicate.

The method expects the following parameters:

  • &$theTemplate: An array reference that will receive the list of templates.
  • $theDatabase: Database connection.
  • $thePredicate: Relationship predicate term object.
  • $theNode: Node to be checked.
  • $theLanguage: Language of labels and definitions.

The provided template array reference will be filled with elements structured as follows:

  • 0: This index will hold the tag global identifier.
  • 1: This index will hold the tag labels of all path elements separated by a carriage return character.
  • 2: This index will hold the tag definitions of all vertex path elements separated by a carriage return character.
  • other: All subsequent elements will hold the respective path term global identifiers.

COntology::_ResolveOntologyNode ( CDatabase  $theDatabase,
SimpleXMLElement  $theElement 
)
protected

Resolve ontology node

This method will parse all elements and items intercepting node attributes, depending on the value of that attribute the method will do the following:

  • term: The value of the element is supposed to be a term global identifier, the method will resolve the master node that is related to that term, replace the attribute value with node and replace the current element value with the native identifier of the resolved master node.
  • node: The method will do nothing, assuming the element's value is a node native identifier.
  • pid: The value of the element is supposed to be a node persistent identifier, the method will resolve the node, replace the attribute value with node and replace the current element value with the native identifier of the resolved node.

If any node is not resolved, the method will raise an exception.

CDatabase$theDatabaseDatabase instance.
SimpleXMLElement$theElementXML element.

COntology::GetTemplateArray (   $theRoot,
  $theLanguage 
)

Retrieve template array

This method can be used to generate an array consisting of the header elements to be used in a template for all subclass elements of the provided node.

The returned array will have as many elements as the tags found in the provided node graph path, each element will be an array featuring in the first item the tag's global identifier, in the second the combined labels of the tag's path terms and in the third the combined definitions of the tag's path; the remaining items will feature the path term global identifiers that can be used for cell merging purposes.

The method expects two parameters:

  • $theRoot: Root node object or reference.
  • $theLanguage: Language of labels and definitions.

The method will traverse the graph using the kPREDICATE_SUBCLASS_OF predicate until it finds a feature node, note that the root node will not be eligible.

If the root is unresolved, the method will return NULL; if no tags are found the method will return an empty array; if the root resolves into more than one node, the method will return a list of templates indexed by the root node identifier.

Parameters
mixed$theRootRoot node reference.
string$theLanguageLanguage code.
