Manage user code

The code, kTAG_USER_CODE, holds the code of the user. This data represents the user native identifier. Once the object has been committed, this attribute will be locked.

The method accepts a parameter which represents either the name or the requested operation, depending on its value:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • other: Set the value with the provided parameter.

The second parameter is a boolean which if TRUE will return the old value when replacing containers; if FALSE, it will return the currently set value.

Manage user domains

The user domain, kTAG_USER_DOMAIN, holds a list of unique values that represent the different domains in which the current user is registered.

This offset collects the list of these domains in an enumerated set that can be managed with the following parameters:

  • $theValue: Depending on the next parameter, this may either refer to the value to be set or to the index of the element to be retrieved or deleted:
    • NULL: This value indicates that we want to operate on all elements, which means, in practical terms, that we either want to retrieve or delete the full list. If the operation parameter resolves to TRUE, the method will default to retrieving the current list and no new element will be added.
    • array: An array indicates that we want to operate on a list of values and that other parameters may also be provided as lists. Note that ArrayObject instances are not considered here as arrays.
    • other: Any other type represents either the new value to be added or the index to the value to be returned or deleted.
  • $theOperation: This parameter represents the operation to be performed whose scope depends on the value of the previous parameter:
    • NULL: Return the element or full list.
    • FALSE: Delete the element or full list.
    • array: This type is only considered if the $theValue parameter is provided as an array: the method will be called for each element of the $theValue parameter matched with the corresponding element of this parameter, which also means that both both parameters must share the same count.
    • other: Add the $theValue value to the list. If you provided NULL in the previous parameter, the operation will be reset to NULL.
  • $getOld: Determines what the method will return:
    • TRUE: Return the value before it was eventually modified.
    • FALSE: Return the value after it was eventually modified.

Load manager object

This method will return the current manager object: if the term is not set, the method will return NULL; if the manager cannot be found, the method will raise an exception.

The object will also be loaded in a data member that can function as a cache.

The method features two parameters: the first refers to the container in which the manager is stored, the second is a boolean flag that determines whether the object is to be read, or if the cached copy can be used.

  $theIdentifier,
  $doThrow = FALSE 
)
static

Resolve a user

This method can be used to locate a user given the attributes that comprise its identifier.

The method accepts the following parameters:

  • $theConnection: This parameter represents the connection from which the terms container must be resolved. If this parameter cannot be correctly determined, the method will raise an exception.
  • $theIdentifier: In this class by default the native identifier is the user code, we assume this parameter is that string.
  • $doThrow: If TRUE, any failure to resolve the term or its namespace, will raise an exception.

The method will return the found user, NULL if not found, or raise an exception if the last parameter is TRUE.

Note: do not provide an array containing the object in the identifier parameter, or you will get unexpected results.

CConnection$theConnectionServer, database or container.
mixed$theIdentifierUser identifier or user reference.
