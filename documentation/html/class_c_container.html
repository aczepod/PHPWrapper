CContainer::ManageObject ( &  $theObject,
  $theIdentifier = NULL,
  $theModifiers = kFLAG_DEFAULT 
)
abstract

Manage an object from the container

This method can be used to manage a single object in the container, it can insert, update, replace, modify, retrieve and delete an object from the current container by providing the object and/or its native unique identifier (kTAG_NID).

This method expects three parameters:

  • $theObject: The object or the data to be handled.
  • $theIdentifier: The native unique identifier (kTAG_NID) of the object. If the value is NULL, it means that it is the duty of the current container to set the value, this will generally be the case when inserting objects; in all other cases the parameter is required.
  • $theModifiers: This parameter represents the operation options, or the list of offsets to be returned. If the parameter is not an array, it will be interpreted as a bitfield where the following values apply:
    • kFLAG_PERSIST_INSERT: The provided object will be inserted in the container. It is assumed that no other element in the container shares the same identifier, if not, the method should raise an exception. In this case the $theObject represents the full object and $theIdentifier represents the object unique identifier, or NULL, if it is the duty of the container to create the identifier. If the identifier was created by the container, it is the duty of this method to set it into the provided object. In this case the method should return the inserted object's native identifier (kTAG_NID).
    • kFLAG_PERSIST_UPDATE: The provided object will replace an object existing in the container. In this case the $theObject represents the full object and $theIdentifier represents the object unique identifier. In this case the method should return TRUE if the object was updated and FALSE if the object was not found.
    • kFLAG_PERSIST_REPLACE: The provided object will be inserted if the identifier is missing or it doesn't match any entry in the container, or it will update the object corresponding to the provided identifier. This option represents the combination of the kFLAG_PERSIST_INSERT and kFLAG_PERSIST_UPDATE operations. In this case the $theObject represents the full object and $theIdentifier represents the object unique identifier, or NULL, if it is the duty of the container to create the identifier. If the identifier was created by the container, it is the duty of this method to set it into the provided object. In this case the method should return the inserted object's native identifier (kTAG_NID).
    • kFLAG_PERSIST_MODIFY: This option can be used to apply modifications to a subset of the object. In this case the parameters of this method have the following functions:
      • $theObject: This parameter will be an array of elements whose key corresponds to the offset to be considered and the value to the value that the operation will use.
      • $theIdentifier: This parameter keeps its original meaning, it should contain the native identifier of the object to be modified. If this parameter is a CQuery instance, this will be used to select the objects to be modified.
      • $theModifiers: Another section of this bitfield will hold the flags that determine what specific modification should be performed:
        • kFLAG_MODIFY_INCREMENT: This flag indicates an increment or decrement operation, the method will take the value corresponding to the offset in $theObject parameter and add it to the object identified by $theIdentifier in the container. For instance, to increment by one the 'A' offset of object X, you would pass the array( 'A' => 1 ) in $theObject, X in $theIdentifier and set the kFLAG_MODIFY_INCREMENT flag; to decrement by two you would pass array( 'A' => -2 ) in $theObject, with all other parameters unchanged. If the offset does not exist, it will be initialised with the increment value.
        • kFLAG_MODIFY_APPEND: This flag indicates that we want to append a value to an existing array, the method will take the value corresponding to the offset in $theObject and append it to the offset in $theObject for the object identified by $theIdentifier in the container. For instance, to append the value 'A' to the array in offset 'FIELD' for object X, you would pass the array( 'FIELD' => 'A' ) in $theObject, X in $theIdentifier and set the kFLAG_MODIFY_APPEND flag. If the field does not exist, it will be created with an array composed of the provided append value; if the field exists and its value is not an array, an exception should be raised. Note that if you provide an array as the value, the method will assume that you want to append the elements of the array as individual items; if you provide an ArrayObject, instead, the method will assume you want to append the whole array as a single item.
        • kFLAG_MODIFY_ADDSET: This flag indicates that we want to add a value to a set, this operation is equivalent to kFLAG_MODIFY_APPEND, except that the value will be appended only if it doesn't exist already in the receiving array.
        • kFLAG_MODIFY_POP: This flag indicates that we want to remove the first or last element from an array. The method will take the value corresponding to the offset in $theObject and check its sign: if positive, the method will remove the first element; if negative it will remove the last. For instance, to remove the first element in the 'FIELD' offset of object X, you would pass the array( 'FIELD' => 1 ) in $theObject, X in $theIdentifier and set the kFLAG_MODIFY_POP flag in $theModifiers; to remove the last element you would pass array( 'FIELD' => -1 ) in $theObject. If the offset does not exist, the method should not fail; if the offset is not an array, the method should raise an exception.
        • kFLAG_MODIFY_PULL: This flag indicates that we want to remove all occurrences of a value from an array. For instance, to remove all occurrances of 'A' from the array contained in the 'FIELD' offset of object X, you would pass array( 'FIELD' => 'A' ) in $theObject, X in $theIdentifier and set the kFLAG_MODIFY_PULL flag in $theModifiers. If the field exists and its value is not an array, an exception should be raised.
        • kFLAG_MODIFY_MASK off: If none of the above flags are set, it means that the key/value pairs set in $theObject represent offsets to be added or removed, depending on the value part of the pair: if the value is NULL, it means that you want to remove the offset; any other value will replace the existing offset or be added to the object.
      In this case the method will return the modified object in the $theObject parameter, or raise an exception if the operation fails.
    • kFLAG_PERSIST_DELETE: This option assumes you want to remove the object from the container. In this case the $theObject represents the full object and $theIdentifier represents the object unique identifier, what counts is that the object's unique identifier (kTAG_NID) is provided. If the object is not found in the container, the method should not fail. In this case the method should return TRUE if the object was deleted and FALSE if the object was not found.
    If none of the above flags are set, it means that the caller wants to retrieve the object identified by the kTAG_NID offset from the provided object or from the provided identifier. If found, the provided object will receive the located object and the method will return TRUE; if not found, the method will set the provided object to NULL and return FALSE. If the parameter is an array, it implies that the requested operation is to retrieve the object, the array will be interpreted as the list of object offsets to be returned, the kTAG_NID offset is returned by default.

The method should raise an exception if the operation was not successful and update the provided object with whatever data the operation will generate.

The method will also raise an exception if the object is not yet initialised (_IsInited()), this happens when the object received the native container.

This class is abstract, which means that derived classes must implement the specific functionality of specialised data stores, for this reason this method is declared abstract.

CContainer::NextSequence (   $theKey,
  $theContainer = NULL 
)

Return a sequence number

This method should return a sequence number connected to the provided key. Each time this method is called, the sequence number is incremented, which means that you should only call it when you intend to use this number.

The first parameter, $theKey, represents the key to the sequence, it will be the kTAG_NID of the record holding the sequence and the field holding the current number has the kOFFSET_SEQUENCE offset.

The second parameter represents the container in which the sequence is stored:

  • string: The method should use the native container corresponding to the provided string.
  • TRUE: The method should use the string stored in kCONTAINER_SEQUENCE_NAME.
  • NULL: The method should use the current container.

If the object is not Inited(), the method should raise an exception.

If the current object does not support sequences, this method should return NULL.

This class is abstract and by default sequences are not supported.

Perform a query

This method can be used to perform a query on the container, it expects an instance of CQuery as the query, or NULL, to query the whole container and two optional parameters that represents the list of desired fields and the list of sort fields with sense.

  • $theQuery: The query expressed as an array or query object, if omitted, it is assumed the query should cover the whole contents of the container.
  • $theFields: The list of fields to be returned, if omitted, it is assumed all fields are to be returned.
  • $theSort: The query sort order provided as an array in which the key represents the field name and he value a number that represents the sense: negative numbers indicate descending order and positive numbers ascending order; a value of zero will be skipped.
  • $theStart: The record number on which to start returning results (zero based).
  • $theLimit: The maximum number of records to be returned.
  • $theResult: This parameter determines the result of the method:
    • NULL: Return the selection cursor (default).
    • TRUE: Return first matched record or NULL if there were no matches.
    • other: Any other value will be interpreted as a field identifier: in this case the method will return an array containing the distinct values of the provided field identifier.

When $theResult is set to NULL, the method should return an object that represents a cursor into the query results, the object should be iterable and feature a method, count, that accepts a boolean parameter: if TRUE the method should return the actual number of elements available taking into consideration paging parameters; if FALSE, the method should return the total affected elements count, that is, the total number of elements affected by the query regardless of paging parameters.

Unserialise provided data element.

This method should convert the provided structure into a custom data type compatible with the current container.

This method is called by a public UnserialiseObject() interface which traverses an object and provides this method with all elements that satisfy the following conditions:

  • CDataType: All instances derived from this class are sent to this method.
  • Array or ArrayObject: If the structure is composed of exactly two offsets and these elements are kTAG_CUSTOM_TYPE and kTAG_CUSTOM_DATA, it will be sent to this method.

The elements to be converted are provided by reference, which means that they have to be converted in place.

This method can also be used in a different way: you can ask the method to convert the provided scalar to the corresponding custom type, for this you need to provide a scalar in the first parameter and a data type in the second.

In this class we declare this class abstract, derived concrete classes must implement it.

CContainer::UnserialiseObject ( &  $theObject)

Unserialise provided object.

This method will convert concrete derived instances of CDataType or equivalent structures into native data types suitable to be stored in containers.

This method will scan the provided object or structure and pass all instances derived from CDataType to another public UnserialiseData() method that will convert these objects into native data types that are compatible with the specific container type.

The method will scan the provided structure and select all elements which are arrays, ArrayObjects or objects derived from CDataType, these elements will be sent to the UnserialiseData() method that will take care of converting these structures into native data types that are compatible with the specific container type.

The method will perform the conversion directly into the provided reference and will use recursion to traverse the provided structures.

Elements sent to the UnserialiseData() method are selected as follows:

  • CDataType: All instances derived from this class are sent to the UnserialiseData() method.
  • Array or ArrayObject: If the structure is composed of exactly two offsets and these elements are kTAG_CUSTOM_TYPE and kTAG_CUSTOM_DATA, it will be sent to the UnserialiseData() method. If the above condition is not satisfied, the structure will be sent recursively to this method.
