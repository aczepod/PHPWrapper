Append statement

This method will append the provided statement to the current query.

Statements of the same type, kOPERATOR_AND and kOPERATOR_NAND, or kOPERATOR_OR and kOPERATOR_NOR, will be added at the same level. If the top level is an kOPERATOR_OR or kOPERATOR_NOR and the provided statement is an kOPERATOR_AND or kOPERATOR_NAND, the latter will be promoted to the top level.

The parameters to this method are:

  • &$theQuery: Query receiving statement.
  • $theCondition: Statement condition.
  • $theCondition: Boolean statement condition code.

Handle condition

This method will chain the provided query to the current query, it expects the first parameter to be a query array and the second parameter to be the condition that will be used to chain the provided query to the current query.

The process of combining queries is governed by a set of three conditions: the current query condition, the provided condition and the statement condition. These conditions are compared as follows:

  • kOPERATOR_OR is equivalent to kOPERATOR_NOR.
  • kOPERATOR_AND is equivalent to kOPERATOR_NAND.

Depending on the combination of these three conditions:

  • current = provided = statement: The provided query statements are recursively sent to the AppendStatement() method using the condition of the provided statement.
  • current = provided: The provided query is appended to the current query.
  • current = statement: A new empty query is created by appending the old current query and the provided query using the provided condition.
  • provided = statement: The current query is appended to the provided query.

CQuery::AppendStatement (   $theStatement,
  $theCondition = NULL 
)

Append statement

This method will append the provided statement or query to the current query.

The first parameter is expected to be either a single statement or a query which is represented by a collection of statements chained by a condition.

The method expects the first parameter to be either an object derived from CQueryStatement, an object derived from this class or an array.

If the current key of the provided statement matches one of the standard conditions (kOPERATOR_AND, kOPERATOR_NAND, kOPERATOR_OR or kOPERATOR_NOR), it is assumed that the provided statement is a query; if that is not the case, it is assumed that the provided statement is a single or collection of statements.

  • Statement: The provided statement will be appended to the current query; see the _HandleStatement() method for more details.
    • Condition provided:
      • Current query is empty: The statement and the condition will become the current query.
      • Current query is not empty: The statement will be appended to the current query using the provided condition.
    • Condition not provided: The condition will be determined:
      • Current query is empty: The condition will default to kOPERATOR_AND.
      • Current query is not empty: The current query condition will be used.
  • Query: The provided query will be chained to the current query; see the _HandleCondition() method for more details.
    • Condition provided:
      • Current query is empty: The provided query will become the current query and the provided condition will be ignored.
      • Current query is not empty: The provided query will be chained to the current query using the provided condition.
    • Condition not provided:
      • Current query is empty: The provided query will become the current query.
      • Current query is not empty: The current query condition will be used.

This method makes use of two protected method for doing the dirty job: _HandleCondition() for queries and _HandleStatement() for statements.

Note: Try not to mix kOPERATOR_OR with kOPERATOR_NOR or kOPERATOR_AND and kOPERATOR_NAND when adding queries, the methods that join queries do not yet handle these combinations well: only mix them when adding statements.
