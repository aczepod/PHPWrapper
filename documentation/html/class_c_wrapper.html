CWrapper::__construct (   $theConnection = NULL,
  $theOptions = NULL 
)

Instantiate class.

The constructor will set-up the environment and parse the request. The workflow is as follows:

  • Check required elements: The method will check if all required elements of the request are there, only if this is the case will the constructor init the service.
  • _InitStatus(): The response status will be initialised to the kSTATUS_IDLE state.
  • _InitOptions(): Service options will be initialised.
  • _InitResources(): Eventual resources are initialised.
  • _ParseRequest(): The request is parsed.
  • _FormatRequest(): The request is normalised if necessary.
  • _ValidateRequest(): The request is validated.

This protected interface should be overloaded by derived classes to implement custom services.

Parameters
mixed$theConnectionNative connection.
mixed$theOptionsConnection options.

CWrapper::_Exception2Status ( Exception  $theException)
protected

Set status from exception.

This method can be used to set the service status according to an exception:

  • CException::Severity(): This value will be set as the status kTERM_STATUS_LEVEL.
  • Exception::getCode(): This value will be set as the status kTERM_STATUS_CODE.
  • Exception::getMessage(): This value will be set in the status kTERM_STATUS_MESSAGE field as a language block.
  • Exception::getFile(): This value will be set in the status kTERM_NOTES.
  • Exception::getLine(): This value will be set in the status kTERM_NOTES.
  • Exception::getTrace(): This value will be set in the status kTERM_NOTES.
  • CException::Reference(): These valuew will be set in the status kTERM_NOTES.

Manage offset.

This method can be used to manage elements within offsets, in other words, it can be used to manage elements within an offset:

  • $theBlock: The main offset.
  • $theElement: The offset within the main offset.
  • $theValue: The new value or the operation:
    • NULL: Retrieve the element in the block.
    • FALSE: Delete the element from the block.
    • other: All other data types are interpreted as a new element.

protected

Parse request.

This method should be used to parse the request, check the request elements and make any necessary adjustments before the request is _ValidateRequest().

This is also where the relevant request elements will be logged to the relative response sections.

The method is called by the constructor and should be overloaded to handle derived classes custom elements.

In this class we handle the kAPI_FORMAT, kAPI_OPERATION and kAPI_STAMP elements.
