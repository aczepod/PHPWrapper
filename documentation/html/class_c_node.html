Manage node description

The node description, kTAG_DESCRIPTION, represents the node's long label or extended definition. It is an optional attribute of the object that holds an array of elements in which the index is represented by the language code and the value is the string.

No two elements may share the same language code.

The description depends on the context in which the object is, as opposed as the definition which does not depend on the context.

The method accepts the following parameters:

  • $theLanguage: Language code.
  • $theValue: The description string or the operation, depending on its value:
    • NULL: Return the string corresponding to the provided language.
    • FALSE: Delete the element corresponding to the provided language.
    • other: Any other value represents the description string that will be set or replace the entry for the provided language.
  • $getOld: If TRUE, the method will return the description string before it was eventually modified; if FALSE, the method will return the value after eventual modifications.

CNode::PID (   $theValue = NULL,

Manage persistent identifier

The persistent identifier, kTAG_PID, holds a string value which represents the object's persistent identifier. This value should uniquely identify the object across implementations and should not change once the object has been committed.

This attribute becomes necessary when the object does not have a global unique identifier and its native identifier is not persistent. In nodes this is true because the Term(), which is its global identifier, is not unique and its native identifier, NID(), is a non persistent sequence number: this attribute can be used to set a value which can be used to differentiate nodes which point to the same term and that may have different native identifiers in different implementations.

The method accepts a parameter which represents either the new value or the requested operation, depending on its type:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • other: Set the value with the provided parameter.

The second parameter is a boolean which if TRUE will return the old value when replacing an existing value; if FALSE, it will return the currently set value.

Note that, while this method allows the creation, modification and deletion of this property, the hosting object may prevent some of these actions, by default this attribute is locked when the object has the _IsCommitted() status.

Create an outgoing edge

This method can be used to instantiate an outgoing edge by providing the object vertex node and edge predicate. The subject vertex is represented by the current node.

Both the predicate and the object vertex node can be any type, the last parameter represents the connection, it is optional and if provided, will be used to commit the edge.

The method expects the following parameters:

  • $theObject: The edge's object vertex.
  • $thePredicate: The edge's predicate.
  • $theConnection: Server, database or container.
  • $doPropagate: This boolean switch indicates whether the relationship is to be propagated. This feature is not used in this class, but derived classes will use it; the default value is TRUE.

The method will return an instance of the CEdge class, if any error occurs, the method will raise an exception.

Manage term synonyms

This method can be used to manage the term's synonyms, kTAG_SYNONYMS, which contains a list of strings that represent alternate codes or names that can be used to identify the term.

This offset collects the list of synonyms in an enumerated set that can be managed with the following parameters:

  • $theValue: Depending on the next parameter, this may either refer to the value to be set or to the index of the element to be retrieved or deleted:
    • NULL: This value indicates that we want to operate on all elements, which means, in practical terms, that we either want to retrieve or delete the full list. If the operation parameter resolves to TRUE, the method will default to retrieving the current list and no new element will be added.
    • array: An array indicates that we want to operate on a list of values and that other parameters may also be provided as lists. Note that ArrayObject instances are not considered here as arrays.
    • other: Any other type represents either the new value to be added or the index to the value to be returned or deleted.
  • $theOperation: This parameter represents the operation to be performed whose scope depends on the value of the previous parameter:
    • NULL: Return the element or full list.
    • FALSE: Delete the element or full list.
    • array: This type is only considered if the $theValue parameter is provided as an array: the method will be called for each element of the $theValue parameter matched with the corresponding element of this parameter, which also means that both both parameters must share the same count.
    • other: Add the $theValue value to the list. If you provided NULL in the previous parameter, the operation will be reset to NULL.
  • $getOld: Determines what the method will return:
    • TRUE: Return the value before it was eventually modified.
    • FALSE: Return the value after it was eventually modified.
