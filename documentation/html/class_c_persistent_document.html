protected

Prepare the object after committing

This method will be called after the object gets committed, its main duty is to update the object after it has been committed.

This method calls three other methods which perform the following functions:

  • _PostcommitRelated(): This method's duty is to update related objects.
  • _PostcommitStatus(): This method's duty is to set the object's status after the persist operation.
  • _PostcommitCleanup(): This method's duty is to cleanup the object after the operation.

These three methods are called in the above order, they take the same arguments as the current method and do not return a value: errors must raise exceptions.

When deriving from this class you should overload the above methods and not this one.

This method accepts two parameters:

  • $theConnection: The container in which the object is to be committed, a server or a database if the DefaultDatabase and DefaultContainer() are able to resolve it into a container.
  • $theModifiers: The commit options provided as a bitfield in which the following values are considered:
    • kFLAG_PERSIST_INSERT: Insert operation.
    • kFLAG_PERSIST_UPDATE: Update operation.
    • kFLAG_PERSIST_REPLACE: Replace operation.
    • kFLAG_PERSIST_DELETE: Delete operation.

Cleanup the object after committing

This method should clean and normalise the object after the operation, it is the last method called by _Postcommit() and the last chance to have a hook into the persist workflow.

Be aware that _PostcommitStatus() has been called before, so if you intend to perform operations that may change the object's status, this is not the place.

The method accepts two parameters:

  • $theConnection: The container in which the object is to be committed, a server or a database if the DefaultDatabase and DefaultContainer() are able to resolve it into a container. This parameter is provided as a reference allowing this method to change it.
  • $theModifiers: The commit options provided as a bitfield in which the following values are considered:
    • kFLAG_PERSIST_INSERT: Insert operation.
    • kFLAG_PERSIST_UPDATE: Update operation.
    • kFLAG_PERSIST_REPLACE: Replace operation.
    • kFLAG_PERSIST_DELETE: Delete operation.
    This parameter is provided as a reference allowing this method to change it.

In this class we reset the _IsDirty() status after committing or loading the object, set the _IsCommitted() status after committing and resetting it after deleting.

In this class we do nothing.

Prepare the object before committing

This method will be called before the object gets committed, its main duty is to check whether the object has all the required elements, to set all default data amd to commit eventual embedded objects.

This method calls three other methods which perform the following functions:

  • _PrecommitValidate(): This method's duty is to check if the object is fot for being committed.
  • _PrecommitRelated(): This method's duty is to check and commit eventual embedded or related objects.
  • _PrecommitIdentify(): This method's duty is to fill the object's identifiers.

These three methods are called in the above order, they take the same arguments as the current method. These methods may return two kind of values:

  • NULL: This value indicates that everything should continue as planned.
  • other: Any other value is interpreted as the object's native identifier, this is an indication that the object was committed and that the calling method can directly return this value.

When deriving from this class you should overload the above methods and not this one.

This method accepts two parameters:

  • $theConnection: The container in which the object is to be committed, a server or a database if the DefaultDatabase and DefaultContainer() are able to resolve it into a container.
  • $theModifiers: The commit options provided as a bitfield in which the following values are considered:
    • kFLAG_PERSIST_INSERT: Insert operation.
    • kFLAG_PERSIST_UPDATE: Update operation.
    • kFLAG_PERSIST_REPLACE: Replace operation.
    • kFLAG_PERSIST_DELETE: Delete operation.

If the conditions for committing the object are not met, an exception should be raised.

Determine the identifiers before committing

This method should set the object identifiers before the current object gets committed. In general it is here where you set sequence numbers or generate unique identifiers.

This method is the last one called by _Precommit(), in general, it should not fail if the first method called by _Precommit() has made the necessary validations. The reason this method is the last in the chain is because often the object identifier may depend on related or embedded objects which have been committed in the previous method.

If this method returns NULL, the caller will continue as usual; if the method returns another type of value, the caller should return that value.

If any error occurs during the process, this method should still raise an exception.

The method accepts two parameters:

  • $theConnection: The container in which the object is to be committed, a server or a database if the DefaultDatabase and DefaultContainer() are able to resolve it into a container. This parameter is provided as a reference allowing this method to change it.
  • $theModifiers: The commit options provided as a bitfield in which the following values are considered:

    • kFLAG_PERSIST_REPLACE: Replace operation.

CPersistentDocument::Delete ( CConnection  $theConnection)

Delete the object from a container

This method will remove the current object from the provided container.

The method expects a single parameter, $theConnection, which represents the container in which the object should be stored. This parameter should be a concrete instance of of CContainer, or a concrete instance of CServer or CDatabase, if the current class has implemented the container resolve interface with DefaultDatabase and DefaultContainer().

Once the object has been deleted, it will have its _IsCommitted() status and its _IsDirty() status reset.

The method will return TRUE if the object was deleted or FALSE if the object was not found in the container.

Note that derived classes should overload the _Precommit() and _Postcommit() methods, rather than overloading this one.

Instantiate an object from a container

This method will retrieve a document from the provided container, instantiate this class with it and return the object; if the document was not located in the container, the method will return NULL.

The method expects a single parameter, $theConnection, which represents the container in which the object should be stored. This parameter should be a concrete instance of of CContainer, or a concrete instance of CServer or CDatabase, if the current class has implemented the container resolve interface with DefaultDatabase and DefaultContainer().

The method expects two parameters:

  • $theConnection: The container in which the object should be stored. This parameter should be a concrete instance of of CContainer, or a concrete instance of CServer or CDatabase, if the current class has implemented the container resolve interface with DefaultDatabase and DefaultContainer().
  • $theIdentifier: The key of the object in the container, by default the kTAG_NID offset.
  • $doThrow: If TRUE, any failure to resolve the object will raise an exception.

If the object could not be located, the method will return NULL.

Note that the resulting object will be of the class with which this static method was called.

CPersistentDocument::NID (   $theValue = NULL,
  $getOld = FALSE 
)

Manage native unique identifier

The native unique identifier, kTAG_NID, represents the primary key of the object in the native format of the container in which the object is stored.

This required property does not have a default data type, it should, however, be of small size, since it represent's the object's primary index.

The method accepts a parameter which represents either the new value or the requested operation, depending on its type:

  • NULL: Return the current value.
  • FALSE: Delete the current value.
  • other: Set the value with the provided parameter.

The second parameter is a boolean which if TRUE will return the old value when replacing an existing value; if FALSE, it will return th currently set value.

Note that, while this method allows the creation, modification and deletion of this property, the hosting object may prevent some of these actions, by default this attribute is locked when the object has the _IsCommitted() status.

Replace the object into a container

This method will insert the current object into the provided container, if not found, or update the object if already there.

CPersistentDocument::Restore ( CConnection  $theConnection)

Restore the object from a container

This method will load the current object with data from a document in a container.

The method expects a single parameter, $theConnection, which represents the container in which the object should be stored. This parameter should be a concrete instance of of CContainer, or a concrete instance of CServer or CDatabase, if the current class has implemented the container resolve interface with DefaultDatabase and DefaultContainer().

The current object must have its native unique identifier offset, kTAG_NID, set, or it must have all the necessary elements in order to generate this identifier.

The method will return TRUE if the operation was successful, NULL if the object is not present in the container, or raise an exception if an error occurs.

If the operation succeeds, the _IsDirty() status will be reset and the _IsCommitted() status will be set.

Note that derived classes should overload the _Postcommit() method, rather than overloading this one, also note that the _Precommit() method is obviously not called here.
